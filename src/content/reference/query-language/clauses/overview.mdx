# Clauses

In SurrealQL, clauses can be used to alter the way a query is executed. They are used in the following ways: 

- [`EXPLAIN`](/docs/reference/query-language/clauses/explain): Explain the query plan.
- [`FETCH`](/docs/reference/query-language/clauses/fetch): Fetch all the fields of related records.
- [`FROM`](/docs/reference/query-language/clauses/from): Specify the table(s) or other target(s) to query from.
- [`GROUP`](/docs/reference/query-language/clauses/group-by): Group the results by a set of fields.
- [`LIMIT`](/docs/reference/query-language/clauses/limit): Limit the number of results.
- [`OMIT`](/docs/reference/query-language/clauses/omit): Omit related records.
- [`ORDER`](/docs/reference/query-language/clauses/order-by): Specify the sort order of the results.
- [`SPLIT`](/docs/reference/query-language/clauses/split): Split the results into a set of subqueries.
- [`START`](/docs/reference/query-language/clauses/start): Return the results of a query starting from a certain point.
- [`WHERE`](/docs/reference/query-language/clauses/where): Specify a condition that acts as a filter.
- [`WITH`](/docs/reference/query-language/clauses/with): Replace the default table iterator with an index iterator.
