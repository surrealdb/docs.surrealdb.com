# Named connections

The SDK config supports multiple connections under the `connections` array. You can target a specific connection for queries, transactions, and schema commands.

> [!WARNING]
> This package depends on the **alpha** v2 SDK. Its API may change.

## Configuring connections

```php
'default' => env('SURREALDB_CONNECTION', 'default'),

'connections' => [
    'default' => [ /* ... */ ],

    'analytics' => [
        'url' => env('SURREALDB_ANALYTICS_URL'),
        'namespace' => env('SURREALDB_ANALYTICS_NAMESPACE'),
        'database' => env('SURREALDB_ANALYTICS_DATABASE'),
        'auto_connect' => false,
    ],
],
```

## Targeting a connection

```php
use SurrealDB\Laravel\Facades\SurrealDB;
use SurrealDB\Laravel\Facades\Surqlize;

// Queries
SurrealDB::run($surql, $bindings, connection: 'analytics');

// A specific client
SurrealDB::client('analytics');

// A scoped executor for Surqlize per-connection execution
$executor = SurrealDB::using('analytics');

// Transactions
Surqlize::transaction($callback, connection: 'analytics');
```

Schema commands accept a `--connection` option:

```bash
php artisan surqlize:schema-apply --connection=analytics
```

> [!NOTE]
> `SurrealDB::using($connection)` returns a scoped `ConnectionQueryExecutor` you can pass to Surqlize for per-connection execution. The flat (single-connection) configuration shape only supports the default connection name.
