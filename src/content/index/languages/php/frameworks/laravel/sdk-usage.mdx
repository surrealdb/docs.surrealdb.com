# SDK usage

You can use the raw [SDK client](/docs/languages/php/v2) directly, or the Laravel manager for lifecycle helpers and named connections.

> [!WARNING]
> This package depends on the **alpha** v2 SDK. Its API may change.

## Raw SDK client

Resolve the `Surreal` client from the container or the `Surreal` facade and use any SDK feature, including the [fluent builders](/docs/languages/php/v2/concepts/fluent-query-builders).

```php
use SurrealDB\SDK\Surreal;

$result = app(Surreal::class)->run('RETURN $message', [
    'message' => 'hello',
]);
```

```php
use SurrealDB\Laravel\Facades\Surreal;

Surreal::run('RETURN true;');
```

## Laravel manager

The `SurrealDB` facade wraps the same SDK client with lifecycle helpers and named-connection support. It lazily connects per query.

```php
use SurrealDB\Laravel\Facades\SurrealDB;

SurrealDB::health();

$result = SurrealDB::run('RETURN $message', [
    'message' => 'hello',
]);
```

Target a specific connection with the `connection:` argument:

```php
$analytics = SurrealDB::run(
    'RETURN $message',
    ['message' => 'hello'],
    connection: 'analytics',
);
```

## HTTP vs WebSocket

The transport is determined by the connection URL. HTTP works for basic queries; live queries, sessions, and native SDK transactions require a WebSocket connection. See [Connecting to SurrealDB](/docs/languages/php/v2/concepts/connecting-to-surrealdb).
