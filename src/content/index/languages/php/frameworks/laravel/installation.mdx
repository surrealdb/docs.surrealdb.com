# Installation

Install the package with Composer and publish its configuration.

> [!WARNING]
> This package depends on the **alpha** v2 SDK and the in-development Surqlize ORM. It requires PHP `8.4` or greater and Laravel `11`, `12`, or `13`.

## Requirements

- PHP `8.4` or greater
- Laravel components `^11.0 || ^12.0 || ^13.0`
- A running [SurrealDB instance](/docs/running/installation)

## Install the package

```bash
composer require surrealdb/laravel
```

Because the SDK and ORM are prerelease, you may need to allow alpha or dev stability. While they are in active development, local workspace installs can use path repositories:

```json
{
    "minimum-stability": "dev",
    "prefer-stable": true,
    "repositories": [
        {
            "type": "vcs",
            "url": "https://github.com/surrealdb/surrealdb.php"
        }
    ]
}
```

## Publish configuration

The package keeps SDK and ORM configuration in separate files. Publish both:

```bash
php artisan vendor:publish --tag=surrealdb-config
php artisan vendor:publish --tag=surqlize-config
```

This creates `config/surrealdb.php` and `config/surqlize.php`.

## Service providers and facades

Laravel auto-discovers the service providers and facade aliases, so no manual registration is required:

- Providers: `SurrealDB\Laravel\SurrealDBServiceProvider`, `SurrealDB\Laravel\SurqlizeServiceProvider`
- Facades: `SurrealDB`, `Surreal`, `Surqlize`
