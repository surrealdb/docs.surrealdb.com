# Surqlize integration

The package wires [Surqlize](/docs/languages/php/orm/surqlize) into Laravel so model queries run through the Laravel-managed SurrealDB connection automatically.

> [!WARNING]
> This package depends on the **alpha** v2 SDK and the in-development Surqlize ORM. Its API may change.

## How it connects

The `SurqlizeServiceProvider` binds the SDK `QueryExecutor` to a deferred executor that resolves `config('surqlize.executor')` (default `surrealdb.connection`) only when a query executes. On boot it registers Surqlize's global `ConnectionManager` with that executor. As a result, ordinary model queries need no manual executor.

## Models

Surqlize models are not Eloquent models. They extend `Surqlize\Model\Model` and use Surqlize attributes.

```php
use Surqlize\Attributes\Id;
use Surqlize\Attributes\Schema;
use Surqlize\Attributes\Table;
use Surqlize\Model\Model;
use SurrealDB\SDK\Types\RecordId;

#[Table('user')]
#[Schema(UserSchema::class)]
final class User extends Model
{
    #[Id]
    public RecordId $id;

    public string $name;
}
```

Register models that should participate in [schema commands](/docs/languages/php/frameworks/laravel/schema-commands) in `config/surqlize.php`:

```php
'models' => [
    App\Models\User::class,
],
```

## Queries

No manual executor is needed for normal usage:

```php
$users = User::select(fn ($user) => [$user->id, $user->name])
    ->where(fn ($user) => $user->name->eq('beau'))
    ->collectModels();

$all = User::query()->collectModels();
```

See [Typed queries](/docs/languages/php/orm/surqlize/typed-queries) for the full builder API.

## Transactions

The `Surqlize` facade exposes a transaction helper that batches ORM queries.

```php
use SurrealDB\Laravel\Facades\Surqlize;

Surqlize::transaction(function ($transaction): void {
    User::createQuery(['name' => 'beau'], executor: $transaction)->execute();
});

Surqlize::transaction(
    fn ($transaction) => User::createQuery(['name' => 'beau'], executor: $transaction)->execute(),
    connection: 'analytics',
);
```

> [!NOTE]
> `Surqlize::transaction()` uses Surqlize's executor-based SurrealQL transaction batching. SDK-native transaction ids are not exposed through the Laravel facade yet, because they depend on WebSocket transport and server feature support. See [Surqlize transactions](/docs/languages/php/orm/surqlize/transactions) and [SDK transactions](/docs/languages/php/v2/concepts/transactions).
