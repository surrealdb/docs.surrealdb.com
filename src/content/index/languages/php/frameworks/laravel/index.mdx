# Laravel

The SurrealDB Laravel integration gives Laravel applications a publishable configuration file, service-container bindings, facades, Artisan schema commands, and testing helpers. Query execution and protocol behavior are delegated to the [v2 PHP SDK](/docs/languages/php/v2); ORM models, query compilation, graph relations, and schema definitions are delegated to [Surqlize](/docs/languages/php/orm/surqlize).

> [!WARNING]
> This package depends on the **alpha** v2 SDK (`surrealdb/surrealdb.php`) and the in-development Surqlize ORM. Expect dependency constraints to tighten as the SDK stabilizes. It requires PHP `8.4` or greater and Laravel `11`, `12`, or `13`.

## What the package provides

- A publishable `config/surrealdb.php` for the SDK client and `config/surqlize.php` for the ORM.
- Service providers and container bindings for the SDK client, a lifecycle manager, and the Surqlize executor.
- `SurrealDB`, `Surreal`, and `Surqlize` facades.
- Artisan commands to dump and apply Surqlize schemas.
- Testing helpers to fake the executor and reset ORM state.
