# Testing

The package provides testing helpers to fake the executor and reset Surqlize's global caches between tests.

> [!WARNING]
> This package depends on the **alpha** v2 SDK. Its testing helpers may change.

## Resetting ORM state

Use the `RefreshSurqlizeState` trait to reset Surqlize's global caches (the connection manager, model metadata, and field-set registry) between tests.

```php
use SurrealDB\Laravel\Testing\RefreshSurqlizeState;

final class UserTest extends TestCase
{
    use RefreshSurqlizeState;

    protected function tearDown(): void
    {
        $this->resetSurqlizeState();

        parent::tearDown();
    }
}
```

The trait also provides `useSurqlizeExecutor($executor)` to set a custom executor, `fakeSurrealDB(?string $connection)` to fake a connection, and `resetSurrealDBFakes()`.

## Faking the executor

Fake the Laravel-managed executor and assert on the queries that were sent.

```php
use SurrealDB\Laravel\Facades\SurrealDB;

$fake = SurrealDB::fake();

SurrealDB::run('RETURN true;');

SurrealDB::assertSurrealQuerySent('RETURN true;');
$fake->assertQuerySent('RETURN true;');
```

Surqlize model queries automatically route through the faked executor while a fake is active.

## Named fakes

Fakes are scoped by connection name.

```php
SurrealDB::fake('analytics');

SurrealDB::run('RETURN true;', connection: 'analytics');

SurrealDB::assertSurrealQuerySent('RETURN true;', connection: 'analytics');
```
