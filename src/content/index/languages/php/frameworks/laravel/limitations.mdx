# Limitations

The Laravel integration is built on the alpha SDK and the in-development Surqlize ORM. This page lists the known limitations and caveats.

> [!WARNING]
> This package depends on the **alpha** v2 SDK. Expect dependency constraints to tighten as the SDK stabilizes.

## Known limitations

- **Alpha dependency chain.** The package depends on the alpha `surrealdb/surrealdb.php` and the in-development `surqlize/surqlize`. Version constraints will tighten over time.
- **Not Eloquent.** Surqlize models extend `Surqlize\Model\Model`, not Laravel's Eloquent model. There is no Eloquent or query-builder integration.
- **No facade for SDK-native transactions.** `Surqlize::transaction()` uses Surqlize's SurrealQL batching. SDK-native transaction ids require the raw `Surreal` client, a WebSocket connection, and a supported SurrealDB version.
- **No wrapper for live queries.** Use the raw SDK over WebSocket. See [Live queries](/docs/languages/php/frameworks/laravel/live-queries).
- **HTTP vs WebSocket.** HTTP works for basic queries; live queries, sessions, and native transactions require WebSocket.
- **Singleton Surqlize connection manager.** Surqlize has one global executor. Use per-query `withExecutor()` or explicit `executor:` arguments when you need multiple executors in the same process.
- **Long-running processes.** Under Octane, queue workers, or long-running commands, connections persist longer than a single request. Tune `disconnect_on_terminate` and the lifecycle settings, and reconnect when needed.
- **Blocking live reads.** `live()` blocks while a subscription is open; plan worker concurrency for SSE or streaming endpoints.
