# Configuration

The package keeps SDK configuration in `config/surrealdb.php` and ORM configuration in `config/surqlize.php`.

> [!WARNING]
> This package depends on the **alpha** v2 SDK. Its configuration may change.

## Environment variables

`config/surrealdb.php` reads these common environment variables:

```dotenv
SURREALDB_CONNECTION=default
SURREALDB_URL=ws://127.0.0.1:8000/rpc
SURREALDB_NAMESPACE=test
SURREALDB_DATABASE=test
SURREALDB_USERNAME=root
SURREALDB_PASSWORD=root
SURREALDB_AUTO_CONNECT=true
SURREALDB_CONNECT_ON_RESOLVE=true
SURREALDB_DISCONNECT_ON_TERMINATE=true
SURREALDB_HEALTH_CHECK_ON_RESOLVE=false
SURREALDB_FORMAT=json
SURREALDB_PING_INTERVAL=30
```

> [!NOTE]
> Use a `ws://` or `wss://` URL when you need live queries, sessions, or native SDK transactions. An `http://` URL works for stateless request/response workloads.

## Named connections

The SDK config supports multiple named connections under a `connections` array, with a `default` connection name.

```php
'default' => env('SURREALDB_CONNECTION', 'default'),

'connections' => [
    'default' => [
        'url' => env('SURREALDB_URL', 'ws://127.0.0.1:8000/rpc'),
        'namespace' => env('SURREALDB_NAMESPACE', 'test'),
        'database' => env('SURREALDB_DATABASE', 'test'),
        'auto_connect' => env('SURREALDB_AUTO_CONNECT', true),

        'lifecycle' => [
            'connect_on_resolve' => env('SURREALDB_CONNECT_ON_RESOLVE', env('SURREALDB_AUTO_CONNECT', true)),
            'disconnect_on_terminate' => env('SURREALDB_DISCONNECT_ON_TERMINATE', true),
            'health_check_on_resolve' => env('SURREALDB_HEALTH_CHECK_ON_RESOLVE', false),
        ],

        'username' => env('SURREALDB_USERNAME'),
        'password' => env('SURREALDB_PASSWORD'),
        'auth' => [
            'mode' => env('SURREALDB_AUTH_MODE'),
            'username' => env('SURREALDB_USERNAME'),
            'password' => env('SURREALDB_PASSWORD'),
            'namespace' => env('SURREALDB_AUTH_NAMESPACE', env('SURREALDB_NAMESPACE', 'test')),
            'database' => env('SURREALDB_AUTH_DATABASE', env('SURREALDB_DATABASE', 'test')),
            'access' => env('SURREALDB_AUTH_ACCESS'),
            'key' => env('SURREALDB_AUTH_KEY'),
            'token' => env('SURREALDB_AUTH_TOKEN'),
            'variables' => [],
        ],

        'driver' => [
            'format' => env('SURREALDB_FORMAT', 'json'),
            'ping_interval' => (int) env('SURREALDB_PING_INTERVAL', 30),
        ],
    ],

    'analytics' => [
        'url' => env('SURREALDB_ANALYTICS_URL'),
        'namespace' => env('SURREALDB_ANALYTICS_NAMESPACE'),
        'database' => env('SURREALDB_ANALYTICS_DATABASE'),
        'auto_connect' => false,
    ],
],
```

See [Named connections](/docs/languages/php/frameworks/laravel/named-connections) for how to target a specific connection.

## Authentication modes

If `SURREALDB_USERNAME` is set, the package uses SDK [`RootAuth`](/docs/languages/php/v2/api/auth/credentials) by default. For scoped authentication, set `SURREALDB_AUTH_MODE` to `namespace`, `database`, `record`, `bearer`, `token`, or `none`, then fill the matching keys in the published config.

## Flat configuration (compatibility)

A flat configuration shape without the `connections` array is also supported for back-compatibility. When both flat keys and a default connection exist, the manager merges the flat keys over the default connection. New applications should prefer the named `connections` form.

## ORM configuration

`config/surqlize.php` holds the executor binding and the ORM model list.

```php
'executor' => env('SURQLIZE_EXECUTOR', 'surrealdb.connection'),

'models' => [
    App\Models\User::class,
],
```
