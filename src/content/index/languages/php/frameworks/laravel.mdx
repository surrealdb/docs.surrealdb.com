# Laravel integration

The [Laravel integration](https://github.com/surrealdb/surrealdb.laravel) connects SurrealDB to a Laravel application. It wires [version 2 of the SDK](/docs/languages/php/v2) and the [Surqlize ORM](/docs/languages/php/ecosystem/surqlize) into Laravel's config, service container, facades, and Artisan commands.

> [!IMPORTANT]
> The integration is published as `0.0.1-alpha.1` and is in early development. It requires PHP `8.4` and Laravel `11`, `12`, or `13`, and depends on the alpha SDK and ORM.

It does not replace Laravel's SQL database. SurrealDB runs alongside your existing connections, and the integration does not use Eloquent or `config/database.php`.

## Installation

```bash
composer require surrealdb/laravel
```

Publish the configuration files for the SDK and the ORM.

```bash
php artisan vendor:publish --tag=surrealdb-config
php artisan vendor:publish --tag=surqlize-config
```

## Configuration

Set the connection details in your environment. The SDK connection lives in `config/surrealdb.php`.

```dotenv
SURREALDB_URL=ws://127.0.0.1:8000/rpc
SURREALDB_NAMESPACE=surrealdb
SURREALDB_DATABASE=docs
SURREALDB_USERNAME=root
SURREALDB_PASSWORD=root
```

Register your Surqlize models in `config/surqlize.php`.

```php
'models' => [
    App\Models\User::class,
],
```

## Using the ORM

Define [Surqlize models](/docs/languages/php/ecosystem/surqlize) and query them directly. The integration registers the executor, so you do not pass one per query.

```php
$users = User::query()->collectModels();

$user = User::create(['name' => 'Tobie', 'age' => 32], id: 'tobie');
```

## Using the SDK facades

The `SurrealDB` facade manages connections and runs raw SurrealQL. The `Surreal` facade resolves the underlying SDK client.

```php
use SurrealDB\Laravel\Facades\SurrealDB;

$result = SurrealDB::run('RETURN $message', ['message' => 'hello']);
```

## Schema commands

Apply or inspect your Surqlize schema with Artisan.

```bash
php artisan surqlize:schema-dump
php artisan surqlize:schema-apply
```

## Testing

The integration provides a fake executor so you can assert the queries your code sends without a live database.

```php
use SurrealDB\Laravel\Facades\SurrealDB;

SurrealDB::fake();
SurrealDB::run('RETURN true;');
SurrealDB::assertSurrealQuerySent('RETURN true;');
```
