# Limitations

Surqlize is in early development. This page lists the known limitations of the current version.

> [!WARNING]
> Surqlize is in early development and depends on the **alpha** [v2 SDK](/docs/languages/php/v2). Its API may change.

## Known limitations

- **Alpha SDK dependency.** Surqlize depends on the alpha `surrealdb/surrealdb.php`. During development a path or VCS repository may be required, and version constraints will tighten as the SDK stabilizes.
- **Compile-first testing.** Most of the test suite is compile-time or mock-executor based. A bundled live-database integration suite is not part of the repository.
- **String APIs are deprecated.** Passing raw string field names to `where()`, `orderBy()`, and related APIs is retained for migration, but typed callbacks are the preferred API.
- **Correlated graph WHERE is deferred.** Graph traversal `WHERE` clauses support field predicates within the traversal segment; correlated outer-table references are deferred to a future version.
- **Model-first RELATE only.** `Model::relate($from)` is model-first only. Edge-first overloads are not supported.
- **No field alias attribute.** There is no `#[Column]`-style alias in v1; PHP property names must match SurrealDB field names.
- **Singleton connection manager.** `ConnectionManager` is a singleton convenience. Use `withExecutor()` for per-query executor injection.

## Validation contracts

Surqlize validates several contracts before building or executing queries:

- Model classes must extend `Surqlize\Model\Model`; edge classes must extend `Surqlize\Edge\Edge`.
- Table and field identifiers are validated before compilation.
- `findOrFail()` throws `Surqlize\Model\Exception\ModelNotFoundException`.
- Persistence methods throw when required `RecordId` values are missing.
- `RELATE` validates edge endpoint classes and endpoint record ids.
