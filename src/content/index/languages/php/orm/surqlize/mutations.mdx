# Mutations

Surqlize exposes both high-level model methods and explicit mutation builders.

> [!WARNING]
> Surqlize is in early development. Its API may change.

## Create

```php
$user = User::create([
    'name' => 'beau',
    'age' => 27,
]);
```

Build the mutation explicitly with `createQuery()`:

```php
$query = User::createQuery(['name' => 'beau', 'age' => 27], id: 'beau');

$query->compile();
// CREATE user:beau CONTENT {"name":"beau","age":27} RETURN AFTER
```

## Update

Update an instance with `save()`:

```php
$user = User::findOrFail('beau');
$user->age = 28;
$user = $user->save();
```

Or update by predicate:

```php
User::updateWhere(fn ($user) => $user->age->gte(18))
    ->merge(['verified' => true])
    ->returnAfter()
    ->execute();
```

## Upsert

```php
$user = User::upsert([
    'name' => 'beau',
    'age' => 27,
], id: 'beau');
```

## Delete

```php
$user = User::findOrFail('beau');
$user->delete();
```

Or delete by predicate:

```php
User::deleteWhere(fn ($user) => $user->age->lt(13))
    ->returnBefore()
    ->execute();
```

## Payloads and return modes

Mutation builders support the following methods.

| Method | Purpose |
|---|---|
| `content($data)` | `CONTENT` payload. |
| `merge($data)` | `MERGE` payload. |
| `replace($data)` | `REPLACE` payload. |
| `patch($patches)` | `PATCH` payload. |
| `returnNone()` | `RETURN NONE`. |
| `returnBefore()` | `RETURN BEFORE`. |
| `returnAfter()` | `RETURN AFTER`. |
| `returnDiff()` | `RETURN DIFF`. |
| `returning($fields)` | Return selected fields. |
| `returningValue($field)` | Return one selected value. |
| `timeout($amount, $unit)` | Add a mutation timeout. |
| `execute()` | Execute and return the raw SDK result. |
| `firstModel()` | Execute and hydrate the first returned model. |

> [!NOTE]
> The string-based `updateWhere()` and `deleteWhere()` helpers accept typed callbacks. Passing raw string field names is retained for migration but typed callbacks are preferred. See [Limitations](/docs/languages/php/orm/surqlize/limitations).
