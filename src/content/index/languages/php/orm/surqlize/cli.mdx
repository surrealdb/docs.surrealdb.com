# CLI

Surqlize ships a Composer binary named `surqlize`.

> [!WARNING]
> Surqlize is in early development. Its API may change.

## Commands

```bash
vendor/bin/surqlize generate:fields [config-path]
vendor/bin/surqlize schema:apply [config-path]
vendor/bin/surqlize memory:footprint [--iterations=1000] [--output=path]
```

In a source checkout without Composer's bin proxy, run the binary directly:

```bash
php bin/surqlize generate:fields surqlize.config.php
php bin/surqlize schema:apply surqlize.config.php
php bin/surqlize memory:footprint
```

## `generate:fields`

Generates typed [field adapters](/docs/languages/php/orm/surqlize/code-generation). The config file expects:

```php
return [
    'models' => [User::class, Address::class],
    'fields_namespace' => 'App\\Models\\Fields',
    'fields_path' => __DIR__ . '/src/Models/Fields',
];
```

## `schema:apply`

Applies [schema definitions](/docs/languages/php/orm/surqlize/schema-dsl). The config file expects `models` plus an SDK query executor:

```php
return [
    'models' => [User::class, Address::class],
    'executor' => $surreal,
];
```

## `memory:footprint`

Generates a JSON memory report covering metadata reflection, field-set resolution, query compilation, hydration, and more.

```bash
vendor/bin/surqlize memory:footprint --iterations=5000 --output=memory-report.json
```

## Composer scripts

| Script | Purpose |
|---|---|
| `composer test` | Run the unit test suite. |
| `composer analyse` | Run PHPStan static analysis. |
| `composer memory` | Run the memory footprint report. |
