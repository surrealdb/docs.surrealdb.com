# Transactions

`ConnectionManager::transaction()` batches bound ORM queries into one transaction. Pass the provided executor to each query through `withExecutor()` or the `executor:` argument.

> [!WARNING]
> Surqlize is in early development. Its API may change.

## Running a transaction

```php
use Surqlize\Connection\ConnectionManager;

ConnectionManager::transaction(function ($transaction): void {
    User::select(['name'])
        ->where(fn ($user) => $user->name->eq('beau'))
        ->withExecutor($transaction)
        ->collect();

    User::createQuery([
        'name' => 'tobie',
        'age' => 30,
    ], executor: $transaction)->execute();
});
```

If the callback throws, the transaction is rolled back and the exception is rethrown.

## Relationship to SDK transactions

This helper collects bound queries during the callback and wraps them in a single `BEGIN TRANSACTION; ... COMMIT TRANSACTION;` batch. It is a different mechanism from the SDK's server-side transaction ids described in [SDK transactions](/docs/languages/php/v2/concepts/transactions), and it works without requiring the server-side transaction feature.
