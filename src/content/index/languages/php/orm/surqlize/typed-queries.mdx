# Typed queries

The Surqlize query builder composes a SurrealQL AST and can either compile a literal query string or execute a bound query through the SDK.

> [!WARNING]
> Surqlize is in early development. Its API may change.

## Compile vs execute

```php
$query = User::select(fn ($user) => [$user->name])
    ->where(fn ($user) => $user->name->eq('beau'));

$literal = $query->compile();       // literal SurrealQL (debugging, tests)
$bound = $query->toBoundQuery();    // SDK BoundQuery (execution)
```

| Method | Result |
|---|---|
| `compile()` | Literal SurrealQL string. |
| `toBoundQuery()` | SDK [`BoundQuery`](/docs/languages/php/v2/api/queries/bound-query). |
| `collect()` | List of raw rows. |
| `collectModels()` | List of hydrated models. |
| `lazyModels()` | Generator of hydrated models. |
| `first()` | First scalar or hydrated model. |

## Typed selects and where clauses

Return one predicate, or an array of predicates combined with `AND`.

```php
User::select(fn ($user) => [$user->name, $user->age])
    ->where(fn ($user) => [
        $user->name->eq('beau'),
        $user->age->gte(18),
    ])
    ->compile();

// SELECT name, age FROM user WHERE name = "beau" AND age >= 18
```

Field helpers include:

| Helper | Operator |
|---|---|
| `eq`, `notEq` | `=`, `!=` |
| `gt`, `gte`, `lt`, `lte` | `>`, `>=`, `<`, `<=` |
| `includes` | `INCLUDES` |
| `contains` | `CONTAINS` |
| `like` | `LIKE` |
| `condition($operator, $value)` | Custom operator |

## Ordering, fetching, and pagination

```php
User::select(fn ($user) => [$user->name])
    ->orderBy(fn ($user) => $user->name->asc())
    ->fetch(fn ($user) => $user->address)
    ->page(page: 3, perPage: 25) // LIMIT 25 START 50
    ->compile();
```

`limit()` and `start()` are available as manual equivalents to `page()`.

## Projections and aggregates

```php
use Surqlize\Query\Fields\Projection;

User::select(fn ($user) => [
        $user->age,
        Projection::count()->as('total'),
    ])
    ->groupBy(fn ($user) => $user->age)
    ->orderBy('total', 'DESC')
    ->compile();

// SELECT age, count() AS total FROM user GROUP BY age ORDER BY total DESC
```

Projection helpers include `Projection::count()`, `Projection::sum($field)`, `Projection::mean($field)`, and `Projection::raw($expression)`.

## Advanced SELECT clauses

```php
User::select(['*'])
    ->omit('password')
    ->withIndex('idx_user_email')
    ->where(fn ($user) => $user->age->gte(18))
    ->split('tags')
    ->orderBy(fn ($user) => $user->name->desc())
    ->limit(10)
    ->start(20)
    ->fetch(fn ($user) => $user->address)
    ->timeout(5)
    ->tempFiles()
    ->explain(full: true)
    ->compile();
```

Other helpers include `withoutIndex()`, `groupAll()`, `withoutFrom()`, and `explainPlan()`.

## Selecting values

```php
$name = User::selectValue(fn ($user) => $user->name)
    ->where(fn ($user) => $user->age->gte(18))
    ->first();
```

`SELECT VALUE` queries return scalar rows and cannot be hydrated with `collectModels()`.

## Search, vector, and geometry

Specialized field helpers compile common SurrealDB search, vector, and geometry expressions.

```php
use Surqlize\Query\Fields\SearchField;
use Surqlize\Query\Fields\VectorField;
use Surqlize\Query\Fields\GeometryField;

$body = new SearchField('body');
SearchableArticle::select(['title', $body->score()->as('score')])
    ->where(fn () => $body->matches('surreal orm'))
    ->orderBy('score', 'DESC')
    ->compile();

$embedding = new VectorField('embedding');
SearchableArticle::select(['title', $embedding->knnDistance()->as('distance')])
    ->where(fn () => $embedding->nearest([0.1, 0.2, 0.3], k: 10, effort: 40))
    ->orderBy('distance')
    ->compile();

$location = new GeometryField('location');
SearchableArticle::select(['*', $location->distanceTo([4.9, 52.3])->as('distance')])
    ->where(fn () => $location->withinMeters([4.9, 52.3], 5000))
    ->orderBy('distance')
    ->compile();
```
