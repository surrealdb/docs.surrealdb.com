# Runtime presets

The v2 SDK can run on different concurrency runtimes. The `Runtime` presets assemble a matching scheduler and transports onto a [`DriverOptions`](/docs/languages/php/v2/api/connection/driver-options) instance, which you pass to `new Surreal(...)`.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Synchronous (default)

The default synchronous runtime works everywhere with no extra dependencies.

```php
use SurrealDB\SDK\Surreal;
use SurrealDB\SDK\Runtime\Runtime;

$db = new Surreal(Runtime::sync());
// equivalent to new Surreal()
```

## OpenSwoole

The OpenSwoole preset runs the SDK on coroutines and uses OpenSwoole's native WebSocket client. Enabling runtime hooks makes the default PSR-18 HTTP transport non-blocking. Use it inside a coroutine context (for example `Co::run()`).

```php
$db = new Surreal(Runtime::swoole());
```

Requires the `openswoole/core` package and the OpenSwoole extension.

## Amp / Revolt

The Amp preset runs on Revolt fibers using non-blocking Amp WebSocket and HTTP transports. It requires a running event loop, such as FrankenPHP worker mode.

```php
$db = new Surreal(Runtime::amp());
```

Requires `revolt/event-loop`, `amphp/websocket-client`, and `amphp/http-client`.

## Presets at a glance

| Preset | Scheduler | Best for |
|---|---|---|
| `Runtime::sync()` | Synchronous | PHP-FPM, CLI scripts, simple usage |
| `Runtime::swoole()` | OpenSwoole coroutines | High-concurrency OpenSwoole apps |
| `Runtime::amp()` | Revolt fibers | FrankenPHP worker mode, Amp apps |

## Customizing a preset

Each preset accepts an existing `DriverOptions`, so you can combine a runtime with other driver configuration:

```php
use SurrealDB\SDK\Connection\DriverOptions;
use SurrealDB\SDK\Enum\CodecEnum;

$options = new DriverOptions(format: CodecEnum::CBOR);

$db = new Surreal(Runtime::swoole($options));
```
