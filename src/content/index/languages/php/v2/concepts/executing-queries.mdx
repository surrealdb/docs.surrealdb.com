# Executing queries

The v2 SDK gives you two ways to run SurrealQL directly: `run()` for raw SurrealQL with bindings, and `query()` for a prebuilt [bound query](/docs/languages/php/v2/concepts/bound-queries). Both return one result per statement. For a higher-level, type-safe API, see the [fluent query builders](/docs/languages/php/v2/concepts/fluent-query-builders).

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Running raw SurrealQL

`run()` executes a SurrealQL string with optional bindings and returns a list of results, one entry per statement.

```php
$result = $db->run('SELECT * FROM type::table($tb)', [
    'tb' => new Table('person'),
]);

// Multiple statements return one entry each.
[$created, $people] = $db->run(
    'CREATE person SET name = $name; SELECT * FROM person;',
    ['name' => 'Tobie'],
);
```

Bindings are sent as query parameters, so values are never concatenated into the query text. Typed [SurrealDB values](/docs/languages/php/v2/concepts/codecs-and-typed-values) such as `RecordId`, `Table`, `Datetime`, and `Duration` are preserved.

## Running a bound query

`query()` executes a [`BoundQuery`](/docs/languages/php/v2/api/queries/bound-query), which is the compiled form produced by the fluent builders. This is useful when you build a fragment programmatically.

```php
use SurrealDB\SDK\Query\BoundQuery;

$query = new BoundQuery('SELECT * FROM person WHERE age > $min', ['min' => 18]);

$result = $db->query($query);
```

Every fluent builder can be compiled into a `BoundQuery` with `->compile()`, so you can inspect or reuse the generated SurrealQL:

```php
$bound = $db->select(new Table('person'))->where('age > 18')->compile();

$result = $db->query($bound);
```

## Error handling

Each statement result is unwrapped from the response; a failed statement throws a [`ServerException`](/docs/languages/php/v2/api/errors). See [Error handling](/docs/languages/php/v2/concepts/error-handling) for the full exception hierarchy.
