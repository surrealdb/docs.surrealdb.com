# Connection lifecycle

The v2 SDK manages connection state, automatic reconnection, lifecycle events, and a middleware pipeline.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Connection status

The connection moves through the `ConnectionStatus` states: `Disconnected`, `Connecting`, `Reconnecting`, and `Connected`.

```php
use SurrealDB\SDK\Connection\ConnectionStatus;

if ($db->status() === ConnectionStatus::Connected) {
    // ready
}

$db->isConnected(); // shorthand for the Connected state
```

## Subscribing to events

`subscribe()` registers a listener for a high-level connection event and returns an unsubscribe closure. The available events are `connecting`, `connected`, `reconnecting`, `disconnected`, `error`, `auth`, and `using`.

```php
$unsubscribe = $db->subscribe('connected', function (string $version): void {
    // connected to a server reporting $version
});

$unsubscribe(); // stop listening
```

## PSR-14 events

In addition to the string events above, the SDK dispatches typed PSR-14 events to a configured event dispatcher, including `Connecting`, `Connected`, `Disconnected`, `Reconnecting`, `ConnectionError`, `AuthChanged`, `NamespaceDatabaseSelected`, `RpcRequestSent`, `RpcResponseReceived`, and `LiveMessageReceived`. Provide a dispatcher through [`DriverOptions`](/docs/languages/php/v2/api/connection/driver-options):

```php
use SurrealDB\SDK\Surreal;
use SurrealDB\SDK\Connection\DriverOptions;

$db = new Surreal(new DriverOptions(events: $dispatcher));
```

## Reconnection

By default the SDK reconnects with exponential backoff when a connection drops. Configure it through the `reconnect` option on [`ConnectOptions`](/docs/languages/php/v2/api/connection/connect-options): pass `false` to disable, an array of settings, or a custom reconnect strategy. On reconnect, the SDK restores the selected namespace and database, session variables, and re-applies authentication.

## Middleware

The SDK runs RPC calls through a middleware pipeline that includes authentication retry, logging, and retry middleware. You can append your own middleware through `DriverOptions`.

## Version and feature gating

On connect, the SDK reads the server version and checks it against its supported range (`1.0.0` up to but not including `4.0.0`). Feature-gated operations such as sessions, transactions, and refresh tokens additionally require SurrealDB `3.0.0` or later and are validated at call time. Set `versionCheck: false` on `ConnectOptions` to skip the version check.
