# Bound queries

A `BoundQuery` is a parameter-bound SurrealQL fragment: the query text plus the values bound to generated placeholders. It is the minimal contract the networking layer relies on, and it is what the [fluent builders](/docs/languages/php/v2/concepts/fluent-query-builders) produce.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Creating a bound query

Pass query text and a bindings array to the constructor:

```php
use SurrealDB\SDK\Query\BoundQuery;

$query = new BoundQuery('SELECT * FROM person WHERE age > $min', ['min' => 18]);

$result = $db->query($query);
```

## Binding values incrementally

Use `bind()` to bind a value and receive a generated `$placeholder`, and `append()` to add SurrealQL or merge another `BoundQuery`. Generated placeholders are named `bind_0`, `bind_1`, and so on.

```php
$query = new BoundQuery('SELECT * FROM person');

$placeholder = $query->bind(18);          // "$bind_0"
$query->append(" WHERE age > {$placeholder}");
```

`append()` guards against duplicate binding keys and throws an `ExpressionException` if a key collides.

## Why bindings matter

Bindings keep values out of the query text, which avoids SurrealQL injection and preserves type information. Typed [SurrealDB values](/docs/languages/php/v2/concepts/codecs-and-typed-values) passed as bindings are serialized in their type-preserving form.

> [!NOTE]
> SurrealQL identifiers such as table names and record ids cannot be sent as parameters. The value types escape them inline instead. The builders handle this for you when you pass a `Table` or `RecordId` as a statement target.
