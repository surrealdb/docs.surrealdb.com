# Fluent query builders

The v2 SDK provides fluent builders for the common SurrealQL statements. Each builder method on `Surreal` returns a builder object. Configure it with chained modifiers, then call `->execute()` to run it, or `->compile()` to produce a [`BoundQuery`](/docs/languages/php/v2/concepts/bound-queries).

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Targets and `ONLY`

Builders accept a `RecordId`, a `Table`, or a raw string as the statement target. When the target is a single `RecordId`, the SDK emits `ONLY`, so `->execute()` returns a single record. When the target is a `Table`, it returns a list.

```php
use SurrealDB\SDK\Types\RecordId;
use SurrealDB\SDK\Types\Table;

$one  = $db->select(RecordId::from('person', 'tobie'))->execute(); // single record
$many = $db->select(new Table('person'))->execute();               // list of records
```

## Execute vs compile

```php
$builder = $db->select(new Table('person'))->where('age > 18');

$records = $builder->execute(); // run through the bound executor
$bound   = $builder->compile(); // inspect the generated BoundQuery
```

`execute()` returns the single statement's result (or `null`). A builder created through `Surreal` is already bound to an executor; you can also construct builders with an explicit executor.

## SELECT

`select()` returns a `SelectQuery` with field projection, filtering, paging, fetching, timeout, and version modifiers.

```php
$people = $db->select(new Table('person'))
    ->fields('name', 'age')   // SELECT type::fields([...])
    ->where('age > 18')        // WHERE
    ->start(0)                 // START
    ->limit(10)                // LIMIT
    ->fetch('account')         // FETCH record links
    ->timeout('5s')            // TIMEOUT
    ->execute();

// Single field value (SELECT VALUE)
$names = $db->select(new Table('person'))->value('name')->execute();
```

## CREATE

```php
use SurrealDB\SDK\Enum\Output;

$created = $db->create(RecordId::from('person', 'tobie'))
    ->content(['name' => 'Tobie', 'visits' => 1])
    ->output(Output::AFTER)
    ->execute();
```

`CreateQuery` supports `content()`, `patch()`, `output()`, `timeout()`, and `version()`.

## UPDATE and UPSERT

`UpdateQuery` and `UpsertQuery` support `content()`, `merge()`, `replace()`, `patch()`, `where()`, `output()`, and `timeout()`.

```php
$db->update(RecordId::from('person', 'tobie'))
    ->merge(['visits' => 2])
    ->execute();

$db->upsert(RecordId::from('person', 'tobie'))
    ->content(['name' => 'Tobie'])
    ->execute();
```

## DELETE

`DeleteQuery` defaults to `RETURN BEFORE`, so deleted records are returned. It supports `output()`, `timeout()`, and `version()`.

```php
$deleted = $db->delete(RecordId::from('person', 'tobie'))->execute();
```

## INSERT

`insert()` accepts records directly, or a target `Table` plus records. `InsertQuery` supports `relation()` (`INSERT RELATION`), `ignore()` (`INSERT IGNORE`), `output()`, `timeout()`, and `version()`.

```php
$db->insert(new Table('person'), [
    ['name' => 'Tobie'],
    ['name' => 'Jaime'],
])->execute();
```

## RELATE

`relate()` creates graph edges. Pass single record ids, or arrays of record ids to relate many at once (the edge must be a `Table` in that case). `RelateQuery` supports `content()`, `unique()`, `output()`, `timeout()`, and `version()`.

```php
$edge = $db->relate(
    RecordId::from('person', 'tobie'),
    new Table('likes'),
    RecordId::from('article', 'surrealdb'),
)->content(['strength' => 10])->execute();
```

## Calling functions

`call()` invokes a SurrealQL or SurrealML function (`fn::*`, `ml::*`, or built-ins). Arguments are bound as parameters.

```php
$greeting = $db->call('fn::greet', args: ['Tobie'])->execute();
```

## Output modifier

The `output()` modifier accepts the `Output` enum: `Output::NONE`, `Output::NULL_`, `Output::DIFF`, `Output::BEFORE`, and `Output::AFTER`.
