# Live queries

Live queries push real-time notifications when records change. They are only available over a WebSocket connection.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

> [!IMPORTANT]
> Live queries require a WebSocket connection (`ws://` or `wss://`). On the HTTP engine, the SDK throws an `UnsupportedFeatureException`.

## Starting a live query

Start a live query by running a `LIVE SELECT` statement. The server returns a live query id (a UUID).

```php
$uuid = $db->run('LIVE SELECT * FROM person')[0];
```

## Consuming notifications

Subscribe to a live query by id with `live()`, which returns an iterable of [`LiveMessage`](https://github.com/surrealdb/surrealdb.php) objects. Iterating reads notifications as they arrive.

```php
use SurrealDB\SDK\Live\LiveAction;

foreach ($db->live($uuid) as $message) {
    $message->queryId;  // the live query id
    $message->action;   // LiveAction enum
    $message->record;   // the affected record id
    $message->value;    // the record payload

    if ($message->action === LiveAction::Delete) {
        // handle removal
    }
}
```

The `LiveAction` enum has the cases `Create`, `Update`, `Delete`, and `Killed`.

> [!WARNING]
> Iterating a live query blocks the PHP process while it waits for the next notification. If your PHP runtime serves requests from a single worker, that one blocked process freezes the server and the entire application becomes unusable until the stream ends. Always run the PHP runtime with multiple workers (for example a multi-worker `php artisan serve`, PHP-FPM with several workers, [Laravel Octane](https://laravel.com/docs/octane), or FrankenPHP), use an async [runtime preset](/docs/languages/php/v2/concepts/runtime-presets) (OpenSwoole or Amp), or run live consumers in a dedicated process. For web applications, see the [Laravel live query patterns](/docs/languages/php/frameworks/laravel/live-queries).

## Events

When a live notification is received, the SDK also dispatches a PSR-14 `LiveMessageReceived` event, which you can observe through a configured event dispatcher. See [Connection lifecycle](/docs/languages/php/v2/concepts/connection-lifecycle).
