# Error handling

The v2 SDK throws typed exceptions for both client-side and server-side failures. All exceptions extend a common base class so you can catch broadly or narrowly.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Base exception

All SDK exceptions extend `SurrealDB\SDK\Exceptions\SurrealException`.

```php
use SurrealDB\SDK\Exceptions\SurrealException;

try {
    $db->select(new Table('person'))->execute();
} catch (SurrealException $error) {
    // any SDK error
}
```

## Server-side exceptions

Errors reported by the server are raised as a `ServerException` (or a specific subclass) carrying a structured `kind`, `message`, `code`, and `details`.

| Exception | Typical cause |
|---|---|
| `ValidationException` | Invalid input rejected by the server |
| `QueryException` | A SurrealQL statement failed |
| `ThrownException` | A `THROW` in SurrealQL |
| `NotFoundException` | A referenced resource was not found |
| `AlreadyExistsException` | A uniqueness or existence constraint failed |
| `NotAllowedException` | The operation was not permitted |
| `SerializationException` | The server could not serialize a value |
| `ConfigurationException` | Invalid configuration |
| `InternalException` | An internal server error |

## Client-side exceptions

These are raised by the SDK before or around a request.

| Exception | Typical cause |
|---|---|
| `AuthenticationException` | Authentication failed |
| `ConnectionUnavailableException` | No active connection |
| `HttpConnectionException` | HTTP transport failure |
| `UnsupportedEngineException` | Unknown URL scheme |
| `UnsupportedFeatureException` | The engine lacks a feature (for example live queries on HTTP) |
| `UnavailableFeatureException` | The server version is too old for a feature |
| `UnsupportedVersionException` | The server is outside the supported range |
| `MissingNamespaceDatabaseException` | Namespace or database not selected |
| `InvalidSessionException` | Invalid session id |
| `ReconnectExhaustionException` | Reconnection retries exhausted |
| `CallTerminatedException` | The connection closed mid-call |
| `ExpressionException` | Invalid query fragment |
| `InvalidValueException` | An invalid typed value |
| `UnexpectedServerResponseException` | A protocol mismatch |

## Catching specific errors

```php
use SurrealDB\SDK\Exceptions\UnsupportedFeatureException;
use SurrealDB\SDK\Exceptions\ServerException;

try {
    foreach ($db->live($uuid) as $message) {
        // ...
    }
} catch (UnsupportedFeatureException $error) {
    // live queries require a WebSocket connection
} catch (ServerException $error) {
    // a server-side failure
}
```
