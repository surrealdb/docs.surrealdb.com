# Authentication

The v2 SDK uses typed credential classes for authentication. You can authenticate when connecting, or at any time with `signin()`, `signup()`, and `authenticate()`.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Credential types

All credential classes implement the [`Credentials`](/docs/languages/php/v2/api/auth/credentials) interface and live in the `SurrealDB\SDK\Auth` namespace.

| Class | Purpose |
|---|---|
| `RootAuth` | Root (system) user: `new RootAuth($user, $pass)` |
| `NamespaceAuth` | Namespace-scoped user: `new NamespaceAuth($namespace, $user, $pass)` |
| `DatabaseAuth` | Database-scoped user: `new DatabaseAuth($namespace, $database, $user, $pass)` |
| `RecordAccessAuth` | Record access (scope) signin/signup with variables |
| `BearerAuth` | A pre-issued access key |
| `Token` | A raw JWT used with `authenticate()` |
| `Tokens` | An access token and optional refresh token pair |

## Signing in

`signin()` accepts a `Credentials` instance (or an array) and returns a [`Tokens`](/docs/languages/php/v2/api/auth/tokens) object.

```php
use SurrealDB\SDK\Auth\RootAuth;
use SurrealDB\SDK\Auth\DatabaseAuth;

$tokens = $db->signin(new RootAuth('root', 'root'));

$tokens = $db->signin(new DatabaseAuth('test', 'test', 'username', 'password'));

echo $tokens->access; // the JWT access token
```

## Record access (scopes)

Use `RecordAccessAuth` for record access signup and signin. Extra variables are passed through to the access method's `SIGNUP` / `SIGNIN` clauses.

```php
use SurrealDB\SDK\Auth\RecordAccessAuth;

$tokens = $db->signup(new RecordAccessAuth(
    namespace: 'test',
    database: 'test',
    access: 'user',
    variables: [
        'email' => 'user@example.com',
        'pass' => 'secret',
    ],
));

$tokens = $db->signin(new RecordAccessAuth(
    namespace: 'test',
    database: 'test',
    access: 'user',
    variables: [
        'email' => 'user@example.com',
        'pass' => 'secret',
    ],
));
```

## Authenticating with a token

If you already have a JWT, authenticate the connection with `authenticate()`:

```php
use SurrealDB\SDK\Auth\Token;

$db->authenticate(new Token($jwt));
// or
$db->authenticate($jwt);
```

## Authenticating on connect

You can authenticate as part of `connect()` by passing `authentication` to `ConnectOptions`. It accepts a `Credentials` instance, a `Token`, a raw token string, a closure, or an `AuthProviderInterface`. This provider is also used to re-authenticate after a reconnect.

```php
use SurrealDB\SDK\Connection\ConnectOptions;
use SurrealDB\SDK\Auth\RootAuth;

$db->connect('ws://localhost:8000/rpc', new ConnectOptions(
    namespace: 'test',
    database: 'test',
    authentication: new RootAuth('root', 'root'),
));
```

## The currently-authenticated record

Use the `auth()` builder to select the currently-authenticated record (`SELECT * FROM ONLY $auth`):

```php
$me = $db->auth()->execute();
```

## Invalidating a session

```php
$db->invalidate();
```

## Token renewal

When connected over an async runtime, the SDK schedules token renewal before expiry. On the synchronous runtime, renewal is reactive: an auth middleware retries once after re-authenticating when a token has expired. Set `invalidateOnExpiry: true` on `ConnectOptions` to invalidate instead of renewing.
