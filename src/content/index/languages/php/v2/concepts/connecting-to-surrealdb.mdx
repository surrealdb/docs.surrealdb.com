# Connecting to SurrealDB

The v2 SDK connects to SurrealDB over WebSocket or HTTP. The engine is chosen automatically from the URL scheme, and the same `Surreal` interface is used regardless of protocol.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Creating a client

Create a `Surreal` instance, then call `connect()` with an endpoint URL and optional [`ConnectOptions`](/docs/languages/php/v2/api/connection/connect-options).

```php
use SurrealDB\SDK\Surreal;
use SurrealDB\SDK\Connection\ConnectOptions;
use SurrealDB\SDK\Auth\RootAuth;

$db = new Surreal();

$db->connect('ws://localhost:8000/rpc', new ConnectOptions(
    namespace: 'test',
    database: 'test',
    authentication: new RootAuth('root', 'root'),
));
```

To customize engines, codecs, schedulers, or PSR implementations, pass [`DriverOptions`](/docs/languages/php/v2/api/connection/driver-options) to the constructor: `new Surreal($options)`.

## Endpoints and protocols

The URL scheme determines the engine. Remote URLs automatically get a `/rpc` suffix appended when missing, so `http://127.0.0.1:8000` becomes `http://127.0.0.1:8000/rpc`.

| Scheme | Engine | Description |
|---|---|---|
| `ws://`, `wss://` | WebSocket | Long-lived, stateful connection. Supports live queries, sessions, and transactions. |
| `http://`, `https://` | HTTP | Stateless connection. Each request is independent. |

You can also build and pass an [`Endpoint`](/docs/languages/php/v2/api/connection/endpoint) explicitly:

```php
use SurrealDB\SDK\Connection\Endpoint;

$db->connect(Endpoint::parse('http://127.0.0.1:8000'), new ConnectOptions(
    namespace: 'test',
    database: 'test',
));
```

## WebSocket vs HTTP

The two engines differ in the features they support.

| Feature | WebSocket | HTTP |
|---|---|---|
| Queries (`run`, `query`, builders) | Yes | Yes |
| Live queries | Yes | No |
| Sessions (SurrealDB 3.0+) | Yes | No |
| Server-side transactions (SurrealDB 3.0+) | Yes | No |
| Export / import | Yes | Yes |

> [!NOTE]
> On the HTTP engine, `use()`, `let()`, and `unset()` do not maintain server-side session state; session variables are merged into query parameters instead. Use a WebSocket connection when you need live queries, sessions, or server-side transactions.

## Selecting a namespace and database

You can select the namespace and database through `ConnectOptions`, or at any time with `use()`:

```php
$db->use('test', 'test');
```

## Connection status and health

```php
use SurrealDB\SDK\Connection\ConnectionStatus;

$db->status();        // ConnectionStatus enum
$db->isConnected();   // bool
$db->version();       // e.g. "surrealdb-2.1.0"
$db->health();        // throws on failure
```

## Closing a connection

Close the connection when you are done to release resources.

```php
$db->close();
```
