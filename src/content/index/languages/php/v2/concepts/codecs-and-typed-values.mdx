# Codecs and typed values

The v2 SDK serializes data using a wire codec and represents SurrealDB-specific types with dedicated PHP value classes.

> [!WARNING]
> The v2 SDK is an early **alpha**. Its API may change before a stable release.

## Codecs

The codec controls how values are encoded on the wire. Configure it with the `format` option on [`DriverOptions`](/docs/languages/php/v2/api/connection/driver-options).

| Codec | Description |
|---|---|
| `CodecEnum::JSON` | Default. Zero-dependency JSON codec shipped with the SDK. |
| `CodecEnum::CBOR` | Binary codec with the richest type preservation. Requires the CBOR codec dependency. |

```php
use SurrealDB\SDK\Surreal;
use SurrealDB\SDK\Connection\DriverOptions;
use SurrealDB\SDK\Enum\CodecEnum;

$db = new Surreal(new DriverOptions(format: CodecEnum::CBOR));
```

Over the JSON protocol, typed values use a tagged representation (for example `{"$uuid": "..."}`) to preserve type information; CBOR preserves types natively.

## Typed value classes

The value classes live in the `SurrealDB\SDK\Types` namespace and extend a common `Value` base. Pass them as bindings or builder data and they round-trip with their SurrealDB type intact.

| Type | Class | Example |
|---|---|---|
| Record id | `RecordId`, `StringRecordId`, `RecordIdRange` | `RecordId::from('person', 'tobie')` |
| Table | `Table` | `new Table('person')` |
| UUID | `Uuid` | `Uuid::v7()` |
| Datetime | `DateTime` | `DateTime::now()` |
| Duration | `Duration` | `Duration::fromString('1h30m')` |
| Decimal | `Decimal` | `new Decimal('19.99')` |
| Bytes | `Bytes` | `Bytes::fromBase64($b64)` |
| Range | `Range`, `BoundIncluded`, `BoundExcluded` | `new Range($begin, $end)` |
| Set | `Set` | `new Set([1, 2, 3])` |
| Geometry | `GeometryPoint`, `GeometryLine`, `GeometryPolygon`, and more | `new GeometryPoint(12.0, 49.0)` |
| Other | `None`, `Future`, `File` | `new None()` |

```php
use SurrealDB\SDK\Types\RecordId;
use SurrealDB\SDK\Types\DateTime;
use SurrealDB\SDK\Types\Duration;

$db->create(RecordId::from('event', 'launch'))
    ->content([
        'at'  => DateTime::now(),
        'ttl' => Duration::fromString('24h'),
    ])
    ->execute();
```

## Escaping vs serialization

Each value type can be rendered two ways:

- `escape()` produces a SurrealQL literal that is safe to inline into query text. The builders use this for identifiers such as table names and record ids, which cannot be sent as parameters.
- `jsonSerialize()` produces the tagged JSON form used for bound parameters.
