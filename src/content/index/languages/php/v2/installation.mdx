# Installation

This guide walks you through installing the v2 alpha SDK into your project with Composer.

> [!WARNING]
> The v2 SDK is an early **alpha** (`v2.0.0-alpha.0`) under the `SurrealDB\SDK` namespace. It requires PHP `8.4` or greater. For production use today, use the [stable v1 SDK](/docs/languages/php/v1/installation).

## Requirements

- PHP `8.4` or greater
- [Composer](https://getcomposer.org/download/)
- A running [SurrealDB instance](/docs/running/installation) (`2.x` or newer recommended)

## Install the SDK

Because v2 is a prerelease, you may need to allow alpha stability for this package. Require the alpha release explicitly:

```bash
composer require surrealdb/surrealdb.php:^2.0@alpha
```

Alternatively, set your project's minimum stability so the alpha can be resolved, then require the package:

```json
{
    "minimum-stability": "alpha",
    "prefer-stable": true
}
```

```bash
composer require surrealdb/surrealdb.php
```

> [!NOTE]
> While the package is in alpha, exact version constraints may change between releases. Pin to a specific alpha tag if you need reproducible installs.

## Optional runtime dependencies

The default synchronous runtime works out of the box. The SDK uses [PSR](https://www.php-fig.org/psr/) HTTP, event, and log interfaces, and ships a zero-dependency JSON codec. For other runtimes and transports, install the matching suggested packages:

| Use case | Packages |
|---|---|
| Default HTTP engine (PSR-18 client and PSR-17 factories) | `guzzlehttp/guzzle`, `php-http/discovery` |
| OpenSwoole coroutine runtime | `openswoole/core` |
| Amp / Revolt runtime (for example FrankenPHP worker mode) | `revolt/event-loop`, `amphp/websocket-client`, `amphp/http-client` |

See [Runtime presets](/docs/languages/php/v2/concepts/runtime-presets) for how to select a runtime.

## Import the SDK

Include the Composer autoloader and import the `Surreal` class:

```php
require __DIR__ . '/vendor/autoload.php';

use SurrealDB\SDK\Surreal;
```
