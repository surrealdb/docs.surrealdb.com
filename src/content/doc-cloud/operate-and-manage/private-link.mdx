# AWS PrivateLink

## Overview

[AWS PrivateLink](https://docs.aws.amazon.com/vpc/latest/privatelink/concepts.html) provides private connectivity between your VPC and SurrealDB Cloud without exposing traffic to the public internet. Traffic routed through AWS PrivateLink stays within the AWS network, which can provide lower latency, help improve security posture, and potential egress cost savings.

> [!IMPORTANT]
> PrivateLink is an **enterprise feature**. Contact [SurrealDB Sales](https://surrealdb.com/contact) to enable PrivateLink for your organization.

## Prerequisites

Before setting up AWS PrivateLink for Surreal Cloud, ensure you have the following:

- A Surreal Cloud organization
- An AWS account with a VPC in a [supported region](#supported-regions)
- IAM permissions to create and manage VPC Endpoints in your AWS account

Please contact [SurrealDB Sales/Support](https://surrealdb.com/contact) and provide your AWS Account ID and Surreal Cloud Organization to enable PrivateLink for your organization

## Supported regions

PrivateLink is available in the following AWS regions. Use the **VPC Endpoint Service Name** when creating a VPC endpoint in AWS.

| AWS Region Name | AWS Region | VPC Endpoint Service Name |
| ----------- | ----------- | ------------------------- |
| US West 2 (Oregon) | `us-west-2` | `com.amazonaws.vpce.us-west-2.vpce-svc-0383e386bbcafdc69` |

> [!NOTE]
> Additional regions may be added over time. If you need PrivateLink in a region not listed here, contact [SurrealDB support](https://surrealdb.com/contact).

## Create a VPC Endpoint in AWS

Create a VPC endpoint in your AWS account to establish the private connection.

1. Open the [Amazon VPC Console](https://console.aws.amazon.com/vpc/) and navigate to **Endpoints** in the left sidebar.

3. Endpoint Settings
3a. Enter a descriptive name for the endpoint (e.g., `surreal-cloud`).
3b. For Type, select the **Endpoint services that use NLBs and GWLBs** category

4. Service Settings
4a. In the **Service name** field, enter the VPC Endpoint Service Name from the [supported regions table](#supported-regions) that matches your instance's region.
4b. Click **Verify service** to confirm the service is found.

5. Network Settings
5a. Select the **VPC** where your application workloads run.
5b. **Under Additional Settings** select **Enable private DNS name**
5c. Select the **subnets** in which to create the endpoint network interfaces. Choose subnets in multiple availability zones for high availability.

## Configure DNS

To resolve your instance's `private_host` to the VPC endpoint, you need to configure DNS.

### Option 1: Enable private DNS (recommended)

If the VPC endpoint service supports private DNS, you can enable it directly on the endpoint:

1. In the VPC Console, select your endpoint.
2. Click **Actions** → **Modify private DNS name**.
3. Enable **Private DNS names**.
4. Click **Save changes**.

With private DNS enabled, your instance's `private_host` hostname automatically resolves to the VPC endpoint's private IP addresses within your VPC.



## Access types

When PrivateLink is enabled for your organization, you can configure each instance with one of three access types:

| Access type | Description | `host` (public endpoint) | `private_host` (PrivateLink endpoint) |
| ----------- | ----------- | ------------------------ | ------------------------------------- |
| **Public** | Default. Instance is accessible via the public internet only. | Available | Not available |
| **Private** | Instance is accessible only through PrivateLink. No public endpoint is exposed. | Not available | Available |
| **Dual** | Instance is accessible via both public internet and PrivateLink. | Available | Available |

- **Public** is the default for all instances and does not require PrivateLink to be enabled.
- **Private** removes the public endpoint entirely, ensuring all traffic flows through PrivateLink.
- **Dual** gives you both endpoints, useful during migration or when you have clients both inside and outside AWS.

> [!NOTE]
> When using the **Private** or **Dual** access type, connect to your instance using the `private_host` value shown in the Surrealist dashboard.





## Enable PrivateLink for an instance

You can set the access type when creating a new instance or change it on an existing instance from the Surrealist dashboard.

### Set access type on a new instance

1. Navigate to the **SurrealDB Cloud** dashboard in Surrealist.
2. Click **Create Instance**.
3. In the instance configuration form, locate the **Access type** setting.
4. Select **Private** or **Dual** from the dropdown.
5. Complete the remaining instance configuration and click **Create**.

{/* TODO: Add screenshot of instance creation form with access type dropdown */}

### Change access type on an existing instance

1. Open the **SurrealDB Cloud** dashboard in Surrealist.
2. Click on the instance you want to modify.
3. Click the **Configure instance** button to open the instance configuration drawer.
4. Change the **Access type** to your desired setting (**Public**, **Private**, or **Dual**).
5. Click **Apply** to save your changes.

> [!NOTE]
> Changing the access type will briefly put your instance into an **Updating** state. The instance remains available during this transition, but the change may take a few moments to propagate.

Once the access type is set to **Private** or **Dual**, the dashboard displays a `private_host` value for your instance. You will need this value to configure DNS and verify connectivity.


## Verify connectivity

After creating the VPC endpoint and configuring DNS, verify the connection from a resource within your VPC.

### DNS resolution check

From an EC2 instance or container in your VPC, verify that the `private_host` resolves to a private IP address:

```bash
nslookup <your-instance-private-host>
```

The response should return private IP addresses (e.g., `10.x.x.x` or `172.x.x.x`), not public IPs.

### Connection test with curl

```bash
curl -v https://<your-instance-private-host>/health
```

A successful response confirms that your VPC can reach the SurrealDB Cloud instance through PrivateLink.

### Connection test with SurrealDB CLI

```bash
surreal sql --endpoint https://<your-instance-private-host> --namespace test --database test --username root --password <your-password>
```

### Connection test with SDK

```javascript
import Surreal from "surrealdb";

const db = new Surreal();
await db.connect("https://<your-instance-private-host>/rpc");
await db.signin({ username: "root", password: "<your-password>" });
```

## Limitations

- PrivateLink is available in [select AWS regions](#supported-regions) only.
- PrivateLink must be enabled at the **organization level** by the SurrealDB team before you can configure instance access types.
- Changing the access type briefly puts the instance into an **Updating** state.
- VPC endpoints incur [AWS charges](https://aws.amazon.com/privatelink/pricing/) in your AWS account.
- Cross-region PrivateLink is not supported — your VPC endpoint must be in the same AWS region as your SurrealDB Cloud instance.
