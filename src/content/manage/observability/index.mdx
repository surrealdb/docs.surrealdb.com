# Observability overview

A SurrealDB database can be monitored in a number of ways:

- [Logging](/docs/build/deployment/observability/logging) — server log level, text or JSON format, files, sockets, and slow-query logging (no metrics stack required).
- [Prometheus (pull)](/docs/build/deployment/observability/observability) — scrapers call `GET /metrics`; authentication boundaries for sensitive series; how OpenTelemetry names map to Prometheus.
- [OTLP (push)](/docs/build/deployment/observability/telemetry) — the process pushes metrics, audit and slow-query logs, and traces to a collector when `SURREAL_TELEMETRY_PROVIDER=otlp` is set.
- [Tokio console](/docs/build/deployment/observability/tokio-console) — optional Tokio runtime debugging (tasks, poll times). Useful for deep performance work alongside metrics and traces.

Prometheus (pull) means a scraper calls your server (typically `GET /metrics`). OTLP (push) means your server opens an export connection and sends metrics, logs, and traces to a collector.

For diagrams, full metric tables, and migration lists maintained with the engine, see the [OBSERVABILITY.md](https://github.com/surrealdb/surrealdb/blob/main/doc/OBSERVABILITY.md) and [TELEMETRY.md](https://github.com/surrealdb/surrealdb/blob/main/doc/TELEMETRY.md) files in the [SurrealDB GitHub repository](https://github.com/surrealdb/surrealdb).
