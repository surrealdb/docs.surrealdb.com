# Cloud management

This section covers the day-to-day management of a SurrealDB Cloud deployment. If you are setting up Cloud for the first time, start with the [getting started guide](/docs/build/deployment/surrealdb-cloud/getting-started) and [connecting](/docs/build/deployment/surrealdb-cloud/connecting) pages instead.

<img src="~/assets/img/image/cloud/light/cloud-architecture-light.png" darkSrc="~/assets/img/image/cloud/cloud-architecture-dark.png" alt="SurrealDB Cloud topology: a managed instance endpoint backed by a compute node and centralised storage on Amazon S3." />

Topics covered here include:

- [Instance management](/docs/manage/cloud/instance-management) — creating, pausing, resuming, and deleting instances.
- [Scaling](/docs/manage/cloud/scaling) — adjusting compute and storage to match your workload.
- [Network access](/docs/manage/cloud/network-access) — IP allowlisting, VPC peering, and PrivateLink configuration.
- [Backups & recovery](/docs/manage/cloud/backups-and-recovery) — automated backups, retention policies, and point-in-time restore.
- [Monitoring & logs](/docs/manage/cloud/monitoring-and-logs) — dashboards, metrics, and log access.
- [Organisations & users](/docs/manage/cloud/organisations-and-users) — team management, roles, and invitations.
- [Billing & support](/docs/manage/cloud/billing-and-support) — plans, invoices, and support channels.
- [AWS Marketplace](/docs/manage/cloud/aws-marketplace) — subscribing to SurrealDB Cloud through AWS Marketplace.
