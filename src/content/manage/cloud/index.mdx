# Cloud management

This section covers the day-to-day management of a SurrealDB Cloud deployment. If you are setting up Cloud for the first time, start with the [getting started guide](/docs/build/deployment/surrealdb-cloud/getting-started) and [connecting](/docs/build/deployment/surrealdb-cloud/connecting) pages instead.

## Cloud plans

SurrealDB Cloud offers managed deployments from development through production-scale clusters:

| Plan | Topology | Storage | Best for |
| --- | --- | --- | --- |
| **Start** (incl. free tier) | Single-node | Dedicated storage per instance | Development, staging, and workloads that scale vertically |
| **Scale** | Multi-node cluster | [SurrealDS](https://surrealdb.com/platform/surrealds) distributed storage | Fault-tolerant, horizontally scalable production |

Start instances scale by resizing CPU, memory, and storage on one node. Scale clusters add query nodes and use SurrealDS for replication, consensus, and distributed transactions — without you operating the cluster stack yourself.

For architecture diagrams and plan comparison, see [Cloud architecture](/docs/build/deployment/surrealdb-cloud/architecture) and [Deployment](/docs/build/deployment). For pricing, see [surrealdb.com/pricing](https://surrealdb.com/pricing).
