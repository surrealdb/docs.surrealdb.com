# Monitoring & logs

Operating production databases requires visibility into **health**, **capacity**, and **errors**. SurrealDB Cloud exposes monitoring and logging so you can detect issues early and audit activity without running your own observability stack for the data plane.

## Monitoring dashboard

The **Cloud monitoring** experience surfaces instance status, resource usage, and key database signals in one place. Use it for day-to-day checks before you escalate to external APM or log platforms.

## Metrics

Typical **metrics** include CPU and memory utilisation, storage consumption, connection counts, and request or query throughput—exact sets depend on instance type and version. Watch sustained high utilisation or rapid storage growth as signals to **scale** or optimise workloads.

## Logs

**Logs** capture operational events, authentication and access patterns where available, and errors from the database engine. **Retention** defines how long logs remain queryable in Cloud; export or ship logs to your SIEM if you need longer archival for compliance.

Configure alerts (where supported) so on-call teams are notified when thresholds breach—for example, error rate spikes or disk nearing capacity.
