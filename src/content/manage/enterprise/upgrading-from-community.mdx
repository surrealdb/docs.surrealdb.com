# Upgrading from Community

Upgrading to **SurrealDB Enterprise Edition** is primarily a **deployment change**: you replace the Community binary or container image with the EE build that matches your version line, then supply a valid **licence key** and complete **activation** as described in [Licensing & activation](/docs/manage/enterprise/licensing-and-activation).

**Data compatibility** is preserved when you stay on the same major and minor release family: your existing storage directories, export files, and replication topology remain valid. Always take a **backup** before switching images, and test failover paths if you run a cluster.

After upgrade, **Community features** continue to behave as before. **Enterprise-only capabilities**—such as distributed live queries, audit logging, FIPS mode, trusted execution, and integrated file storage—become available once the server reports an active EE licence; enable each feature deliberately via configuration and security review.

If you need a staged rollout, run EE in a non-production environment first, validate application queries and drivers, then promote the same image and licence process to production during a maintenance window.
