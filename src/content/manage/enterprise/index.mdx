# Installation

SurrealDB Enterprise Edition (EE) is distributed separately from the Community Edition. After your organisation purchases EE, you receive download credentials or a private registry path for the build that matches your platform and architecture (Linux packages, container images, or other supported artefacts).

At a glance:

1. **Download** the Enterprise artefact for your OS and CPU architecture, or pull the private container image with registry credentials from your welcome pack.
2. **Install** the binary or deploy the image using the same operational patterns you use for other stateful services (systemd, Kubernetes, and so on).
3. **Configure** a **licence key** and any cluster or compliance settings before traffic reaches the instance.

During installation, place your **licence key** where the server can read it—typically an environment variable or a secured file path referenced by configuration. Do not commit licence material to source control or bake it into public images.

On first start, the server performs **activation** against SurrealDB’s licensing service unless you are using an **offline activation** bundle (see [Licensing & activation](/docs/manage/enterprise/licensing-and-activation)). Activation confirms entitlement and may bind the key to identifiers you provide (for example hostname or cluster id, depending on your agreement).

After activation, verify the deployment by checking the server logs for a successful licence check and by running a minimal connectivity test from your application tier. For operational tuning, continue with the same configuration surface as Community Edition, then enable EE-only features as required.
