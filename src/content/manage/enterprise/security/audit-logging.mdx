# Audit logging

**Audit logging** in SurrealDB Enterprise Edition records **security-relevant events** so you can demonstrate who did what, and when—for example authentication attempts, privilege changes, administrative actions, and access to sensitive resources. The exact event catalogue can vary by release; refer to your EE release notes for the authoritative list.

Logs are emitted in a **structured format** suitable for parsing (commonly JSON lines or another documented encoding) so you can ship them to central storage without ad hoc string scraping. Include timestamps in UTC, stable event types, and correlation identifiers where available.

**Storage** is typically your organisation’s log pipeline: forward to syslog, object storage, or a managed log service with retention and immutability controls that match your policy.

For **SIEM integration**, ingest the audit stream into Splunk, Elastic, Microsoft Sentinel, or similar platforms using their HTTP, syslog, or object-storage collectors. Map SurrealDB fields to your canonical schema so alerts, dashboards, and investigations stay consistent across services.
