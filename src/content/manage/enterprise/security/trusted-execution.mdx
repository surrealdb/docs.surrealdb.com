# Trusted execution

**Trusted execution environments (TEEs)** provide hardware-isolated regions where code and data are protected from the host operating system and other processes, subject to CPU vendor guarantees. SurrealDB Enterprise Edition can integrate with TEEs when you need stronger boundaries around key material or sensitive query processing.

**Hardware requirements** depend on the TEE technology you choose (for example Intel TDX, Intel SGX, or AMD SEV—availability varies by CPU generation and cloud instance type). Plan capacity so enclave memory limits and attestation overheads remain within your latency budget.

**Attestation** allows a remote party to verify that the correct, unmodified SurrealDB build is running inside a genuine enclave before secrets are released. Your security architecture should define who performs attestation (your control plane, a KMS, or a dedicated service) and how often it repeats.

Deploy TEE features only on supported platforms with firmware and microcode current per your vendor’s security bulletins; TEEs reduce attack surface but do not replace network controls, [Audit logging](/docs/manage/enterprise/security/audit-logging), or sound key management.
