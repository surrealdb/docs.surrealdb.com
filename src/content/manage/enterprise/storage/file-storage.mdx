# File storage / S3 / blob

SurrealDB Enterprise Edition extends the database with **file and blob storage** so large binaries live alongside structured records without ad hoc object stores for every use case. Backends are typically **S3-compatible**—AWS S3, MinIO, or other providers that speak the same API—giving you durable, off-node storage with your existing access policies.

In SurrealQL, **`DEFINE BUCKET`** (and related statements for your release) lets you declare **buckets** that map to storage locations, enforce access rules, and expose files through the same security model as the rest of your database. Application code can upload, fetch, and reference blobs while keeping metadata queryable in tables.

Operationally, treat buckets like any other external dependency: configure **credentials** via IAM roles or keys rotated by your secrets manager, enable **encryption** at rest and in transit, and monitor usage and costs per bucket.

For compliance-heavy environments, combine this feature with [Audit logging](/docs/manage/enterprise/security/audit-logging) and your object-store access logs so you retain a complete trail of who touched which objects.
