# Sessions and turns

Every interaction between an agent and a user happens inside a **session**. Sessions and turns are first-class objects in Spectron — not implementation details. Understanding them is essential to understanding how memory is created, where it comes from, and how to audit it later.

## What is a session?

A session is a conversation container. It has a stable identifier, carries scope (the user, agent, and organisation the conversation belongs to), and acts as the unit of provenance for all memory that is produced from it.

Sessions are opened explicitly, receive turns, and are closed when the conversation ends. All memory extracted during a session traces back to that session's identifier.

```python
import spectron

memory = spectron.Memory(api_key="sk-…")

session = await memory.sessions.create(
    scope={"org": "acme", "user": "u_alice", "agent": "support"},
)

print(session.id)   # sess_01hw…
print(session.scope)  # {"org": "acme", "user": "u_alice", "agent": "support"}
```

```javascript
import { Memory } from "spectron";

const memory = new Memory({ apiKey: "sk-…" });

const session = await memory.sessions.create({
    scope: { org: "acme", user: "u_alice", agent: "support" },
});

console.log(session.id);    // sess_01hw…
console.log(session.scope); // { org: "acme", user: "u_alice", agent: "support" }
```

## What is a turn?

A turn is a single message within a session. Each turn has a **role** that identifies who produced it:

| Role | Description |
|-----------|----------------------------------------------|
| `user` | A message from the human participant |
| `assistant` | A message generated by the AI model |
| `system` | A system prompt or instruction injected by the application |
| `tool` | The output of a tool call made by the assistant |

Turns are appended in order. The sequence of turns in a session forms the full conversation history.

```python
await session.add_turn(role="user", content="My name is Alice and I work at Acme Corp.")
await session.add_turn(role="assistant", content="Hello Alice! How can I help you today?")
await session.add_turn(role="user", content="I need to reset my API key.")
```

```javascript
await session.addTurn({ role: "user", content: "My name is Alice and I work at Acme Corp." });
await session.addTurn({ role: "assistant", content: "Hello Alice! How can I help you today?" });
await session.addTurn({ role: "user", content: "I need to reset my API key." });
```

## The extraction pipeline

Every turn addition triggers the extraction pipeline automatically. You do not need to call any separate "process" step.

When a turn is added, Spectron:

1. Identifies entities, attributes, and relations mentioned in the new turn
2. Reconciles new facts against existing memory (applying authority and temporal logic)
3. Persists stable facts into Layer 1
4. Links each persisted fact back to the source turn via `source_turn`

This happens asynchronously in the background. For most applications, the pipeline completes within a few hundred milliseconds. If you need to wait for pipeline completion before querying memory, use the `await_processing` flag:

```python
await session.add_turn(
    role="user",
    content="My name is Alice and I work at Acme Corp.",
    await_processing=True,
)
```

```javascript
await session.addTurn(
    { role: "user", content: "My name is Alice and I work at Acme Corp." },
    { awaitProcessing: true },
);
```

## Provenance via source_turn

Every memory fact carries a `source_turn` field. This is the identifier of the turn from which the fact was extracted. `source_turn` makes memory fully auditable: you can trace any entity or attribute back to the exact message that introduced it.

```python
profile = await memory.context(scope={"org": "acme", "user": "u_alice"})

for entity in profile.entities:
    print(entity.label, "→", entity.source_turn)
    # alice → turn_01abc…
    # acme_corp → turn_01abc…
```

```javascript
const profile = await memory.context({ scope: { org: "acme", user: "u_alice" } });

for (const entity of profile.entities) {
    console.log(entity.label, "→", entity.sourceTurn);
    // alice → turn_01abc…
    // acme_corp → turn_01abc…
}
```

You can retrieve the original turn content at any time:

```python
turn = await memory.sessions.get_turn(turn_id="turn_01abc…")
print(turn.role, turn.content)
```

## Session lifecycle

A session moves through three phases:

**Open** — the session is active and accepts new turns. Memory extraction runs on every turn.

**Closed** — the session is marked complete. No new turns can be added. Memory persists and is available for recall.

**Archived** — optional long-term state. The session record is retained for audit but session-scoped context memories may be subject to TTL expiry depending on your retention policy.

```python
# Create
session = await memory.sessions.create(scope={…})

# Add turns over time
await session.add_turn(role="user", content="…")
await session.add_turn(role="assistant", content="…")

# Close when done
await session.close()

# Retrieve later (read-only)
session = await memory.sessions.get("sess_01hw…")
print(session.status)  # "closed"
```

```javascript
// Create
const session = await memory.sessions.create({ scope: { … } });

// Add turns over time
await session.addTurn({ role: "user", content: "…" });
await session.addTurn({ role: "assistant", content: "…" });

// Close when done
await session.close();

// Retrieve later (read-only)
const closed = await memory.sessions.get("sess_01hw…");
console.log(closed.status); // "closed"
```

## Sessions as the unit of provenance

Scope determines which agents and users have access to which memories. But **sessions** determine where those memories came from. When a fact is later recalled, disputed, or needs to be corrected, the `source_turn` chain lets you follow it back to:

- The session in which it was stated
- The turn within that session
- The role that produced that turn (user, assistant, tool, or system)

This makes Spectron's memory layer fully traceable. You know not just *what* the agent knows, but *when* it learned it and *from whom*.

For details on how the extraction pipeline reconciles conflicting facts and applies authority rules, see [Extraction pipeline](/spectron/memory/reasoning-model/extraction-pipeline) and [Authority hierarchy](/spectron/memory/reasoning-model/authority-hierarchy).
