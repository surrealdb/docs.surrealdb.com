# LoCoMo

LoCoMo is a long-conversation question-answering benchmark published at ACL 2024 by Snap Research. Spectron runs it as part of its nightly accuracy evaluation.

## What it measures

LoCoMo contains **10 long conversations** of 300+ turns each and **1,540 questions** that require retrieving and reasoning over memory stored during the conversation. Questions span five categories:

| Category | Description |
|---|---|
| Single-hop factual | Questions answerable from one recalled fact |
| Multi-hop reasoning | Questions that require combining two or more facts |
| Temporal reasoning | Questions about when events happened or in what order |
| Open-domain | General knowledge questions grounded in conversation context |
| Adversarial / not-mentioned | Questions whose answers were never stated; correct answer is "I don't know" |

## Evaluation method

Spectron evaluates LoCoMo using **LLM-as-judge** with generous partial credit. The harness:

1. Creates a fresh Spectron session and ingests all conversation turns in order, exercising the real extraction and reconciliation pipeline.
2. For each question, calls `resolve_query()` and formats the retrieved memory hits plus the question into a synthesis prompt.
3. Passes the question, expected answer, and candidate answer to the LLM judge.
4. Aggregates verdicts into per-category accuracy and an overall score.

Special tolerances:
- **Adversarial questions** accept "not mentioned", "I don't know", or paraphrases as correct.
- **Temporal questions** allow ±1 day/week tolerance for date-based answers.

The published dataset has a known issue: approximately 6.4% of ground truth labels are incorrect. Spectron's evaluation notes this in the benchmark report.

## Reference scores

The LoCoMo leaderboard provides context for Spectron's scores:

| System | LoCoMo score |
|---|---|
| ByteRover (SOTA) | 92.2% |
| Full-context ceiling | 87.52% |

The full-context ceiling reflects a system that injects the entire conversation into the LLM context window rather than using memory retrieval. It sets an upper bound on what extraction-based approaches can achieve.

## Running LoCoMo

The benchmark runs via the `spectron-eval` CLI (included in the Spectron server build):

```bash
# Full accuracy run (requires real LLM API keys)
spectron-eval bench locomo

# Limit to the first 3 conversations and 100 questions
spectron-eval bench locomo --max-items 3 --max-questions 100

# Save a Markdown report
spectron-eval bench locomo --report ./reports/locomo-$(date +%Y%m%d).md

# Dry run – exercises the pipeline without writing to the database
spectron-eval bench locomo --no-write

# CI mode – uses deterministic fake LLM and embeddings (no API keys needed)
spectron-eval bench locomo --fake
```

The `--fake` mode exercises the pipeline plumbing (session creation, turn ingestion, query resolution) using a deterministic fake LLM. It does not produce meaningful accuracy numbers but is used in per-PR CI to detect regressions in the ingestion pipeline.

## CI strategy

**Per-PR (no API keys required):** A 1-item smoke test with `--fake` ensures the pipeline compiles and runs end-to-end. It diffs against the committed baseline at `eval/bench/BASELINE-locomo.md`.

**Nightly (real LLMs):** The full benchmark runs with real LLM providers via `bench-live.yml`. Reports are uploaded as CI artifacts. No committed baselines are used for real-LLM runs because LLM output is non-deterministic.

## Feature flag

LoCoMo is gated behind the `bench-locomo` feature flag in the Rust build:

```toml
[features]
bench-locomo = []
bench = ["bench-locomo", "bench-statebench", "bench-longmemeval"]
```

To build with all benchmarks:

```bash
cargo build --features bench
```
