# Cookbooks

The cookbooks are practical, end-to-end guides for building with Spectron. They assume familiarity with the [eight pillars](/docs/spectron/architecture/eight-pillars-and-categories) and [unified substrate](/docs/spectron/mental-model/two-layer-architecture) and focus on implementation decisions rather than introductory explanation.

## Build

Step-by-step guides for common agent types:

- [Customer support agent](/docs/spectron/cookbooks/build/customer-support-agent) – persistent customer memory with authoritative knowledge product knowledge and per-user experiential memory history.
- [Personal AI assistant](/docs/spectron/cookbooks/build/personal-ai-assistant) – preferences, biographical facts, and behavioural instructions that persist across sessions.

## Patterns

Reusable integration patterns that apply across many agent types:

- [User memory in chat](/docs/spectron/cookbooks/patterns/user-memory-in-chat) – the canonical pattern for adding per-user memory to a chat application.
- [Knowledge-grounded agents](/docs/spectron/cookbooks/patterns/knowledge-grounded-agents) – loading authoritative knowledge into authoritative knowledge and letting agents answer from it.
- [Reflection loops](/docs/spectron/cookbooks/patterns/reflection-loops) – using the reflection endpoint to synthesise insights and persist them as new memory.
- [Stateful workflows](/docs/spectron/cookbooks/patterns/stateful-workflows) – tracking long-running multi-step process state in Spectron.
- [Adding memory to an existing app](/docs/spectron/cookbooks/patterns/adding-memory-to-existing-app) – the minimal integration path for retrofitting memory into a running application.

## Migrate

Guides for teams moving from an existing memory solution:

- [From a vector store](/docs/spectron/cookbooks/migrate/from-vector-store) – trading chunk-only retrieval for structured extraction and provenance.
- [From Zep](/docs/spectron/cookbooks/migrate/from-zep) – concept mapping, SDK migration, and scope model differences.
- [From LangMem](/docs/spectron/cookbooks/migrate/from-langmem) – moving from a library-based approach to a server-based one.

## Benchmarks

Evaluation guides and result interpretation:

- [LoCoMo benchmark](/docs/spectron/cookbooks/benchmarks/locomo) – long-context multi-session memory recall and accuracy.
- [StateBench](/docs/spectron/cookbooks/benchmarks/statebench) – state tracking correctness across corrections and temporal transitions.
