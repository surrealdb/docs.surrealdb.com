# Knowledge

Layer 0 is Spectron's authoritative knowledge store. It holds the ground truth loaded from documents, policies, product data, and other organisation-owned content. Where Layer 1 (Memory) learns from conversations, Layer 0 is loaded programmatically and treated as the source of truth the agent relies on.

When Layer 1 memory contradicts Layer 0, Layer 0 wins.

## What belongs in Layer 0

Layer 0 is designed for content that is owned by the organisation and changes on a known schedule:

- Product catalogues and pricing
- Return policies, warranty terms, and service agreements
- Technical documentation and FAQs
- Internal handbooks and onboarding guides
- Data warehouse exports (Snowflake, Databricks)
- Content from collaboration tools (Notion, Confluence, Google Drive, GitHub)

## Key capabilities

**Multi-format ingestion**: Layer 0 accepts text, Markdown, HTML, JSON, PDF, images (PNG, JPEG, WebP, GIF), audio (WAV, MP3, OGG, FLAC, AAC), and video (MP4, WebM, MOV). All parsing runs in-process — no external sidecars or Python runtime.

**Hybrid retrieval**: Four query modes — `vector`, `bm25`, `hybrid` (RRF fusion), and `hybrid_graph` (hybrid plus keyword graph, typed-knowledge graph, section vectors, document links, PMI co-occurrence, and personalised PageRank).

**Keyword graph**: RAKE-extracted keyphrases are stored as nodes with scored edges to their source documents and PMI co-occurrence edges to related keyphrases. The graph powers the `hybrid_graph` density signal and is directly traversable.

**Typed knowledge nodes**: Structured entries (products, policies, categories) that can be imported via the API and linked with typed relation edges. Layer 1 entities resolve to these nodes via `resolves_to`.

**Connectors**: Scheduled sync from external sources (Snowflake, Databricks, Google Drive, Notion, GitHub, Confluence) keeps Layer 0 current without manual uploads.

**Observable pipeline**: Every document has a granular status (`queued → extracting → chunking → embedding → keywording → ready`). Operators can see exactly where each file is in processing.
