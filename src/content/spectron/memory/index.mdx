# Memory

**Memory** workflows cover the **experiential** side of Spectron: sessions, turns, `remember` / `recall` / `reflect` / `forget`, profiles, and tuning. Extracted records still live in the **same** SurrealDB graph as document knowledge – distinguished by provenance and **trust** (`source.kind = "turn"`, reflections, elaborations, consolidations, …) rather than by a separate database.

The experiential side is where “I have a cat”, “I saw a cat last night”, and “I used to have a cat” land as **different kinds** of belief – not one undated note. Start with the six **experiential categories** (including **episodic** raw turns) in [Memory categories](/docs/spectron/mental-model/memory-categories), then [Supersession, decay, and forget](/docs/spectron/mental-model/memory-lifecycle) for how beliefs change over time, and [Traces and memory evolution](/docs/spectron/architecture/traces-and-evolution) for how traces close the loop on ranking and reuse.
