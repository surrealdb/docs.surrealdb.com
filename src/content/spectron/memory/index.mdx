# Memory

Layer 1 is Spectron's conversational memory layer. It holds the structured facts that Spectron extracts from agent sessions: who the user is, what they know, what they are trying to accomplish, what instructions they have given, and what remains uncertain.

Unlike Layer 0 — which stores authoritative knowledge ingested from documents and external systems — Layer 1 is built from conversation. Every session, every turn, every message the agent exchanges with a user contributes to Layer 1. The extraction pipeline runs continuously, reconciling new facts against existing memory and persisting only what is stable and consistent.

## What Layer 1 contains

Layer 1 memory is organised into five categories:

| Category | What it holds |
|--------------|----------------------------------------------------------------|
| Identity | Who the user or agent is: name, role, preferences, traits |
| Knowledge | Facts the user has shared: their domain, their projects, their expertise |
| Context | The current situation: active task, recent focus, temporary state |
| Instructions | Explicit directives given to the agent: how to behave, what to avoid |
| Unknowns | Gaps that have been identified but not yet resolved |

Each category has different persistence characteristics. Identity and Knowledge persist indefinitely. Context expires on a configurable TTL (default seven days). Instructions are kept until explicitly revoked. Unknowns are resolved or promoted to other categories as information arrives.

## Subsections

### Sessions

Sessions are the first-class conversation containers in Spectron. Every turn belongs to a session. Every piece of extracted memory traces back to a source turn. Sessions carry scope and serve as the unit of provenance.

See [Sessions](/spectron/memory/sessions/chat-sessions) for the full reference.

### Operations

The five core memory operations — remember, recall, reflect, forget, and profiles — are the primary interface for reading and writing Layer 1 memory outside of session turns.

- **Remember** — store a fact directly without going through session extraction
- **Recall** — retrieve relevant memories for a query
- **Reflect** — synthesise a coherent summary from accumulated memory
- **Forget** — remove a specific fact or entity
- **Profiles** — fetch a structured snapshot of all known facts for a scope

See [Operations](/spectron/memory/operations/remember) for details on each.

### Reasoning model

The reasoning model describes the rules that govern how Spectron extracts, reconciles, and persists memory. Understanding it explains why Spectron behaves the way it does when facts conflict, when sources disagree, or when memory needs to change over time.

Key topics:

- **Extraction pipeline** — how turns are processed and entities are identified
- **Authority hierarchy** — how conflicting facts from different sources are resolved
- **Reconciliation and supersession** — how new facts update or replace old ones
- **Temporal validity** — how facts carry valid_from/valid_until ranges
- **Instructions and uncertainties** — how directives and unknowns are stored

See [Reasoning model](/spectron/memory/reasoning-model/extraction-pipeline).

### Tuning

Spectron provides several configuration knobs for adjusting how memory behaves in production: caching strategies, model selection per pipeline stage, and ontology grounding for domain-specific extraction.

See [Tuning](/spectron/memory/tuning/models-per-stage).

## Getting started

The quickest way to see Layer 1 in action is to create a session and add a few turns:

```python
import spectron

memory = spectron.Memory(api_key="sk-…")

session = await memory.sessions.create(
    scope={"org": "acme", "user": "u_alice"},
)

await session.add_turn(role="user", content="I'm Alice, a software engineer at Acme Corp.")
await session.add_turn(role="assistant", content="Nice to meet you, Alice.")

profile = await memory.profile(scope={"org": "acme", "user": "u_alice"})
print(profile)
```

After a few turns, `profile` will contain structured facts about Alice: her name, her role, her employer — all extracted automatically, all linked back to their source turns.
