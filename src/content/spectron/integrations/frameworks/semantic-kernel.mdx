# Semantic Kernel

Spectron provides a plugin for [Microsoft Semantic Kernel](https://learn.microsoft.com/en-us/semantic-kernel/), exposing memory recall, storage, and knowledge search as SK kernel functions. The integration is published as a separate package — `Spectron.SemanticKernel` — and is not bundled with the core Spectron SDK.

> **Later-phase integration.** The `Spectron.SemanticKernel` package is planned for a future release and is not yet available. This page describes the intended integration design. Check the [Spectron changelog](/spectron/reference) for availability updates.

## How it works

The Semantic Kernel plugin registers a set of SK functions under the `SpectronMemory` plugin name. These functions are available to the kernel planner and can be invoked directly by agents that need to store or retrieve memory.

| SK function | Description |
|---|---|
| `SpectronMemory.Remember` | Record a fact or conversation turn into memory. |
| `SpectronMemory.Recall` | Retrieve memory relevant to a query. |
| `SpectronMemory.Search` | Search the knowledge base using a natural language query. |
| `SpectronMemory.Forget` | Redact a specific fact or entity from memory. |
| `SpectronMemory.GetProfile` | Retrieve the current attribute profile for a named entity. |

The planner can choose to call these functions as part of a plan step, or you can invoke them directly in your kernel pipeline.

## Installation

### .NET

```bash
dotnet add package Spectron.SemanticKernel
```

## Kernel setup

```csharp
using Microsoft.SemanticKernel;
using Spectron.SemanticKernel;

var kernel = Kernel.CreateBuilder()
    .AddOpenAIChatCompletion("gpt-4o", Environment.GetEnvironmentVariable("OPENAI_API_KEY")!)
    .Build();

var spectronPlugin = new SpectronMemoryPlugin(new SpectronMemoryOptions
{
    Context = "acme-prod",
    ApiKey = Environment.GetEnvironmentVariable("SPECTRON_API_KEY")!,
    Scope = new Scope { Org = "anneal", Agent = "assistant" },
});

kernel.Plugins.AddFromObject(spectronPlugin, "SpectronMemory");
```

### Configuration options

| Option | Type | Description |
|---|---|---|
| `Context` | `string` | Spectron Context identifier |
| `ApiKey` | `string` | Agent-level API key |
| `BaseUrl` | `string?` | Spectron server URL. Defaults to the hosted cloud endpoint. |
| `Scope` | `Scope` | Default scope for memory operations. |
| `SessionId` | `string?` | Existing session to resume. If null, a new session is created. |

## Direct function invocation

```csharp
// Store a fact
await kernel.InvokeAsync("SpectronMemory", "Remember", new KernelArguments
{
    ["role"] = "user",
    ["content"] = "The user's preferred programming language is Rust.",
});

// Retrieve relevant memory
var result = await kernel.InvokeAsync("SpectronMemory", "Recall", new KernelArguments
{
    ["query"] = "What programming languages does the user prefer?",
});

Console.WriteLine(result.GetValue<string>());
```

## Using with the planner

When registered, the kernel planner can autonomously decide to call memory functions as part of a multi-step plan:

```csharp
using Microsoft.SemanticKernel.Planning.Handlebars;

var planner = new HandlebarsPlanner(new HandlebarsPlannerOptions
{
    AllowLoops = true,
});

var plan = await planner.CreatePlanAsync(kernel,
    "Recall what you know about the user and write a personalised greeting.");

var result = await plan.InvokeAsync(kernel);
Console.WriteLine(result);
```

The planner will include a `SpectronMemory.Recall` step automatically when memory retrieval is relevant to the goal.

## Session management

By default, the plugin creates a new Spectron session when first invoked and reuses it for the lifetime of the `SpectronMemoryPlugin` instance. To manage sessions explicitly:

```csharp
// Create a session manually before registering the plugin
var spectronClient = new SpectronClient(new SpectronOptions
{
    Context = "acme-prod",
    ApiKey = Environment.GetEnvironmentVariable("SPECTRON_API_KEY")!,
});

var session = await spectronClient.Sessions.CreateAsync(new Scope
{
    Org = "anneal",
    Agent = "assistant",
    User = userId,
});

var spectronPlugin = new SpectronMemoryPlugin(new SpectronMemoryOptions
{
    Context = "acme-prod",
    ApiKey = Environment.GetEnvironmentVariable("SPECTRON_API_KEY")!,
    SessionId = session.Id, // resume this session
});
```

## Knowledge search

The `SpectronMemory.Search` function queries the knowledge base (Layer 2 — uploaded documents and knowledge nodes) rather than the conversational memory layer (Layer 1):

```csharp
var searchResult = await kernel.InvokeAsync("SpectronMemory", "Search", new KernelArguments
{
    ["query"] = "onboarding process for new employees",
    ["limit"] = 5,
});
```

This is useful when your kernel pipeline needs to ground responses in documents uploaded to Spectron's knowledge layer.
