# Integrations

Spectron integrates with agents through HTTP, MCP, **official SDKs**, and harness adapters that batch conversation turns into `/facts/batch`, using the same provenance and trust model as the server.

The REST surface is described by an OpenAPI specification. Python and TypeScript clients track that spec so request and response shapes stay aligned with the server.

## MCP server (coding assistants)

Native MCP at `/mcp` on the api port — same **`Authorization: Bearer`** auth as REST. Prefer this when the client already speaks MCP (Claude Desktop, Cursor, Windsurf, VS Code, Claude Code).

```bash
spectron mcp
```

→ [MCP server install](/docs/spectron/integrations/mcp-server/install)

## Python SDK

Spectron ships **inside** the main SurrealDB Python package:

```bash
pip install surrealdb
```

```python
from surrealdb import Spectron
```

Spectron ships **inside** the [`surrealdb`](https://pypi.org/project/surrealdb/) package on PyPI — the same install as the SurrealDB Python driver. Do not install the unrelated **`spectron`** package.

→ [Python SDK guide](/docs/spectron/integrations/sdks/python)

## JavaScript and TypeScript SDK

Separate npm package under the SurrealDB scope:

```bash
npm install @surrealdb/spectron
```

→ [JavaScript SDK guide](/docs/spectron/integrations/sdks/javascript-and-typescript)

## Kotlin and Swift

The [`surrealdb.kotlin`](https://github.com/surrealdb/surrealdb.kotlin) and [`surrealdb.swift`](https://github.com/surrealdb/surrealdb.swift) repositories provide **SurrealDB database drivers** (query, auth, live queries). They do not yet include a Spectron memory client — use REST or the Python / TypeScript SDKs until native Spectron packages are added.

## REST API

Direct HTTP from any language. End-user routes: `/api/v1/{context_id}/…`. Management: `/api/v1/contexts/…`.

→ [REST integration guide](/docs/spectron/integrations/surfaces/rest) · [Full reference](/docs/spectron/reference/rest-api)

## Harness adapters (automatic turn capture)

Thin packages that mirror agent runtime traffic into `POST /api/v1/{ctx}/facts/batch` without changing your prompts:

| Platform | Package |
| --- | --- |
| LangChain (Python) | `spectron-langchain` |
| OpenAI Agents (Python) | `spectron-openai` |
| Vercel AI SDK (TS) | `@surrealdb/spectron-vercel-ai` |
| n8n | `n8n-nodes-spectron` |
| Claude Code | `npx install-spectron-hook` |

→ [LangChain](/docs/spectron/integrations/frameworks/langchain) · [Vercel AI SDK](/docs/spectron/integrations/frameworks/vercel-ai-sdk)

## Not shipped yet

- **Semantic Kernel** and **CrewAI** adapters — no packages in the repo; pages are placeholders.
- **Spectron clients for Kotlin / Swift** — database SDKs only today.
- **Embedded in-process library** — use REST or SDK against a deployed instance ([Embedded quickstart](/docs/spectron/index/quickstarts/embedded)).
