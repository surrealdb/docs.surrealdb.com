# Integrations

Spectron exposes four integration surfaces. Choose the one that fits your stack.

## Python SDK

The recommended integration for Python agents and applications. The async client covers sessions, turns, memory operations, knowledge management, and the management API. A synchronous façade is available for codebases that do not use `asyncio`.

```bash
pip install spectron
```

→ [Python SDK guide](/docs/spectron/integrations/sdks/python)

## JavaScript and TypeScript SDK

Full REST parity in JavaScript with TypeScript declarations. ESM and CJS entry points. Every method returns a `Promise`.

```bash
npm install spectron
```

→ [JavaScript SDK guide](/docs/spectron/integrations/sdks/javascript-and-typescript)

## MCP server

Seven high-level tools at `/mcp` — the same port as the REST API, same bearer-token auth. Designed for AI coding assistants that natively speak MCP: Claude Desktop, Cursor, Windsurf, VS Code, and Claude Code.

```bash
npx install-mcp spectron --client cursor --context acme-prod
```

→ [MCP server guide](/docs/spectron/integrations/mcp-server/install)

## REST API

Direct HTTP access for any language or tool. Management endpoints live at `/api/v1/...`; end-user endpoints at `/v1/{context_id}/...`. All responses use JSON; errors follow the RFC 7807 problem details format.

→ [REST API guide](/docs/spectron/integrations/surfaces/rest)

Adapters for LangChain, Semantic Kernel, Vercel AI SDK, and CrewAI are planned for a later phase. They will be published as separate packages.
