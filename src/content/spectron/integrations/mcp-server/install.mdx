# Installing the MCP server

Spectron exposes seven tools over the [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) at the `/mcp` endpoint of the same server that serves the REST API. Any MCP-compatible client – coding assistants, chat interfaces, or your own agent infrastructure – can connect to it using a single bearer token.

## What the MCP server provides

The Spectron MCP server exposes seven tools:

| Tool | What it does |
|---|---|
| `memory_store` | Store a conversational exchange. Spectron auto-classifies, reconciles, and persists entities, attributes, relations, instructions, uncertainties, and corrections. |
| `memory_recall` | Retrieve relevant context for a query from both experiential memory and authoritative knowledge. Returns a formatted context string ready for system prompt injection. |
| `memory_reflect` | Synthesise patterns and insights from existing memories. Optionally persists derived attributes back to experiential memory. |
| `memory_profile` | Return the aggregated profile (identity, preferences, instructions, dynamic context) for the current principal. |
| `knowledge_search` | Search the authoritative knowledge base only. Supports vector, BM25, hybrid, and hybrid-graph modes. |
| `knowledge_get` | Fetch a specific document, chunk, or knowledge node by ID without re-ranking. |
| `memory_forget` | Semantically soft-delete memories matching a natural-language description. |

See [MCP tools reference](/docs/spectron/integrations/mcp-server/tools-reference) for the full input/output schema of each tool.

## One-line install

The `install-mcp` utility writes the Spectron MCP configuration block to the correct config file for each client:

```bash
npx install-mcp spectron --client claude-desktop --context acme-prod
npx install-mcp spectron --client claude-code --context acme-prod
npx install-mcp spectron --client cursor --context acme-prod
npx install-mcp spectron --client vscode --context acme-prod
npx install-mcp spectron --client windsurf --context acme-prod
```

The command prompts for your API key and writes the configuration block shown below. You only need to run it once per client.

## The mcpServers configuration block

Every client uses a variation of the same JSON structure. The block written by `install-mcp` looks like this:

```json
{
  "mcpServers": {
    "spectron": {
      "url": "https://api.spectron.dev/mcp",
      "headers": {
        "Authorization": "Bearer <your-api-key>",
        "X-Spectron-Context": "acme-prod"
      }
    }
  }
}
```

- `url` – the Spectron MCP endpoint. For self-hosted deployments, replace with `https://your-spectron-host/mcp`
- `Authorization` – your Spectron API key as a Bearer token
- `X-Spectron-Context` – the context ID that scopes all tool calls

If you are self-hosting Spectron, replace `https://api.spectron.dev` with your own base URL.

## Scope defaults

Pass `--scope` flags to `install-mcp` to set default scope headers that are included with every tool call. This is useful for coding assistant contexts where the principal is always a specific user within an organisation:

```bash
npx install-mcp spectron \
  --client cursor \
  --context acme-prod \
  --scope user=tobie \
  --scope org=acme
```

This adds an `X-Spectron-Scope` header to the config block:

```json
{
  "mcpServers": {
    "spectron": {
      "url": "https://api.spectron.dev/mcp",
      "headers": {
        "Authorization": "Bearer <your-api-key>",
        "X-Spectron-Context": "acme-prod",
        "X-Spectron-Scope": "user=tobie,org=acme"
      }
    }
  }
}
```

Individual tool calls can still pass a `scope` argument to override or extend the default.

## Authentication

Every MCP tool call is authenticated using the `Authorization: Bearer <token>` header from the config block. The same API keys used for the REST API work for MCP – there is no separate credential system.

If the token is missing or invalid, every tool call returns an error response rather than results.

## MCP vs SDK: when to use each

| Situation | Recommendation |
|---|---|
| Coding assistant (Cursor, VS Code, Windsurf, Claude) | MCP – one-line install, no code changes |
| Production agent in Python or JavaScript | SDK – typed interface, richer error handling, retry logic |
| Custom agent infrastructure or CLI tooling | REST API – language-agnostic, full control |
| Rapid prototyping with any LLM | MCP – the fastest path to memory-enabled agents |

The MCP server and the REST API are backed by the same storage layer. Data written via MCP tools is immediately available through the SDK, and vice versa.
