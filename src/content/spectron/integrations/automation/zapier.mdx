# Zapier

Zapier connects to Spectron over MCP through its **MCP Client** integration, so Zaps and Zapier Agents can call Spectron's memory tools as actions without any custom code. Because each Spectron API key is bound to one Context, the connection needs only a URL and a bearer token.

> [!NOTE]
> Zapier's MCP Client is a beta feature and supports **Streamable HTTP** servers only (SSE is no longer supported). Spectron serves Streamable HTTP MCP at `/mcp`.

## Prerequisites

- A Spectron MCP endpoint and API key. On **SurrealDB Cloud**, use your context host from Surrealist **API keys**; self-hosted, use your Spectron server's base URL.

## Connect

1. In Zapier, add the **MCP Client** integration (search for "MCP Client" when adding an app to a Zap or Agent).
2. Create a connection and provide:
   - **Server URL**: `https://your-context-host/mcp`
   - **Transport**: Streamable HTTP
   - **Bearer token**: your Spectron API key
3. Save the connection. Zapier discovers the Spectron tools and exposes each as an action you can add to a step.

> [!NOTE]
> The API key is bound to one Context, so `context_id` is optional on every tool call. A bearer token is all the MCP Client needs; no extra headers are required.

## Use Spectron in a Zap

Add a Spectron tool as an action step and map fields from earlier steps in the Zap:

- **Store a fact** after a trigger fires, for example when a form is submitted or a support ticket is created, with `memory_store`.
- **Recall context** before an AI step (such as a "Chatbots" or "AI by Zapier" action) with `memory_recall`, then feed the returned context into the prompt.
- **Search knowledge** with `knowledge_search` to ground a reply in synced documents.

A typical support Zap: trigger on a new ticket, call `memory_recall` scoped to the customer, run an AI drafting step with that context, then call `memory_store` to record the resolution.

## Scope per tool call

Narrow reads and writes with the per-tool **`scope`** argument (slash paths, for example `["org/acme/user/alice"]`), mapped from Zap data such as the customer's id. Register paths with `spectron scopes create` before first use.
