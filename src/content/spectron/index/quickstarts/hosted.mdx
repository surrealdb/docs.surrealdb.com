# Hosted quickstart

This guide takes you from a blank terminal to a running memory session against Spectron Cloud. No infrastructure to provision — Spectron Cloud handles the server, SurrealDB instance, and object store for you.

## Step 1 — Sign up

Create an account at [api.spectron.dev](https://api.spectron.dev). Spectron Cloud uses a project-based model: your account can hold multiple projects, each with its own billing and isolation boundary.

## Step 2 — Create a Context

A **Context** is the top-level organisational unit in Spectron. It maps to a single SurrealDB namespace and database pair and owns its own API keys, models configuration, and stored memory.

In the Spectron Cloud dashboard:

1. Click **New Context**.
2. Give it a name, for example `acme-prod`.
3. Choose a region.
4. Click **Create**.

Once the Context is ready, open it and navigate to **API keys**. Create a key, copy it, and keep it safe — you will not be able to view it again.

## Step 3 — Install the SDK

Spectron provides first-party SDKs for Python and JavaScript.

```bash
# Python
pip install spectron

# JavaScript / TypeScript
npm install spectron
```

## Step 4 — Set environment variables

Store your API key in an environment variable so it is not hardcoded in source files.

```bash
export SPECTRON_API_KEY="sk-..."
```

## Step 5 — Store your first memory

### Option A — Spectron drives the loop

Use `session.chat()` when you want Spectron to handle retrieval, LLM generation, and memory storage in a single call. This is the simplest integration path: you provide the user message, Spectron returns the assistant reply, and memory is updated automatically.

```python
import asyncio
import os
from spectron import Spectron

memory = Spectron(
    context="acme-prod",
    base_url="https://api.spectron.dev",
    api_key=os.environ["SPECTRON_API_KEY"],
)

async def main():
    session = await memory.sessions.create(scope={"user": "tobie", "org": "anneal"})

    reply = await session.chat("I just got promoted to CTO")
    print(reply)

    reply = await session.chat("What do you know about me?")
    print(reply)

    await session.close()

asyncio.run(main())
```

```javascript
import { Spectron } from "spectron";

const memory = new Spectron({
  context: "acme-prod",
  baseUrl: "https://api.spectron.dev",
  apiKey: process.env.SPECTRON_API_KEY,
});

const session = await memory.sessions.create({
  scope: { user: "tobie", org: "anneal" },
});

const reply = await session.chat("I just got promoted to CTO");
console.log(reply);

const followUp = await session.chat("What do you know about me?");
console.log(followUp);

await session.close();
```

### Option B — Caller drives the loop

Use `session.turn()` and `session.context()` when you want to control the LLM call yourself. This is the right choice when you are integrating Spectron into an existing agent loop or need to pass the context to a specific model with a custom system prompt.

```python
import asyncio
import os
from spectron import Spectron

memory = Spectron(
    context="acme-prod",
    base_url="https://api.spectron.dev",
    api_key=os.environ["SPECTRON_API_KEY"],
)

async def main():
    session = await memory.sessions.create(scope={"user": "tobie", "org": "anneal"})

    # Store a turn — Spectron extracts entities, attributes, and relations
    result = await session.turn(role="user", content="I just got promoted to CTO")
    print(result.entities)      # extracted entities
    print(result.attributes)    # extracted attributes
    print(result.relations)     # extracted relations
    print(result.corrections)   # facts that contradict stored memory

    # Retrieve formatted context for a system prompt
    context = await session.context(query="What is Tobie's role?")
    print(context.context)      # ready-made string for injection into system prompt

    await session.close()

asyncio.run(main())
```

```javascript
import { Spectron } from "spectron";

const memory = new Spectron({
  context: "acme-prod",
  baseUrl: "https://api.spectron.dev",
  apiKey: process.env.SPECTRON_API_KEY,
});

const session = await memory.sessions.create({
  scope: { user: "tobie", org: "anneal" },
});

const result = await session.turn({ role: "user", content: "I just got promoted to CTO" });
console.log(result.entities);
console.log(result.attributes);

const context = await session.context({ query: "What is Tobie's role?" });
console.log(context.context);

await session.close();
```

## Step 6 — Query memory without a session

For one-off lookups — such as building a system prompt before starting a conversation — use the top-level `context()` method. This does not create a session or store anything.

```python
ctx = await memory.context("brief on tobie", k=10)
print(ctx.context)
```

```javascript
const ctx = await memory.context({ query: "brief on tobie", k: 10 });
console.log(ctx.context);
```

The returned string is formatted for direct injection into a system prompt.
