# Eight pillars and six categories

Spectron’s design is expressed as **primitives** the substrate and pipelines must support – not as a flat checklist of unrelated features.

## Eight pillars of agent memory

Each pillar is something the write path, read path, or storage layer must be able to represent. Together they are the minimum set for memory that holds up under scale, change, and multi-instance deployment.

1. **Authoritative** – curated artefacts: documents, manuals, policies, product data, code, structured exports. Higher default trust because the source is vetted. Encoded as `source.kind = "document"`.
2. **Experiential** – conversational input: what the agent or user said, observed, or was told. Lower default trust than the authoritative stream. Encoded as `source.kind = "turn"`.
3. **Reconciliation** – when streams disagree, new facts do **not** silently overwrite. Cross-provenance conflicts emit explicit **uncertainty** rows. Same-provenance updates form supersession chains (`valid_until` on the prior assertion). **One reconciler** handles document extraction, turn extraction, reflection, and elaboration.
4. **Elaboration** – durable memory by linking knowledge and memories that were not obviously related. Primarily a **background** process; can also run at ingest-time or on-demand. Output carries `source.kind = "elaboration"` and `derived_from` pointing at inputs.
5. **Reflection** – memory minted from the questions the agent is asked. `POST /reflect` runs a synthesis pass; persisted facts use `source.kind = "reflect"` with lower default trust than primary sources.
6. **Consolidation** – background crystallisation of beliefs over time. Pools recent facts, examines observations, and creates, updates (with history), or marks superseded rows. Encoded as `source.kind = "consolidation"`.
7. **Calibration** – every assertion is gated by certainty, provenance, and trust. `attribute.confidence` (reconciler posterior) is stored separately from `source.trust` (source prior). Below a configurable confidence floor, the reconciler refuses to supersede and emits **uncertainty** instead.
8. **Collective** – shared, reconcilable memory across people, agents, and instances. Corroboration policy can promote a fact to a wider scope with `derived_from` pointing at corroborating rows.

## Six memory categories (experiential layer)

The experiential layer is **not** a single bucket. It is six typed sub-stores with distinct schemas, lifecycles, and retrieval behaviours: one raw record and five extracted categories built on top.

| Category | Role |
| --- | --- |
| **Episodic** | The **raw** conversational record: sessions and turns in order, with anaphora intact. Source of truth for lexical attribution (`source.span`). Written once; not reconciled; may age out faster than derived facts. Browsable via session tooling (see [Sessions and turns](/docs/spectron/mental-model/sessions-and-turns)). |
| **Identity** | Durable facts about who the principal is – name, role, employer, preferences. Long retention, low decay, prompt-ready profile. |
| **Knowledge** | Things the principal has learnt or shared, distinct from authoritative documents – projects, observations, references. Medium retention; decays without reinforcement. |
| **Context** | What is going on **now** – active topics, recent intents, the working set for the current conversation. Short retention; replaced rapidly. |
| **Instructions** | **Behavioural** memory, not factual (“always British English”, “never use my first name”). Stored separately; applied at **prompt assembly**, not generic retrieval. |
| **Uncertainty** | Explicit “we do not know yet” rows from the reconciler, cross-provenance clashes, or extraction that cannot ground a reference. Surfaced so gaps are visible instead of hallucinated over. |

A parallel **trace layer** records Spectron’s own retrievals, decisions, and responses – memory **about** how memory was used. It composes with the six categories: traces feed the ranker, calibrator, and consolidator. See [Traces and memory evolution](/docs/spectron/architecture/traces-and-evolution).

For a practitioner-oriented tour of the six extracted categories (and how they appear in APIs), see [Memory categories](/docs/spectron/mental-model/memory-categories).
