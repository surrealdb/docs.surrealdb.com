# Memory categories

Spectron splits the **experiential** side of memory into **six** typed sub-stores. They are **not** interchangeable labels on one table: each has its own lifecycle, decay posture, and retrieval weighting. A parallel **trace layer** records how memory was *used* – see [Traces and memory evolution](/docs/spectron/architecture/traces-and-evolution).

For the eight **pillars** (authoritative, experiential, reconciliation, …) see [Eight pillars and six categories](/docs/spectron/architecture/eight-pillars-and-categories).

## 1. Episodic – the raw conversational record

**Episodic** memory is the ordered **session / turn** stream: what was said, in order, with anaphora intact. It is the substrate every extracted category cites through provenance (`source.ref`, `source.span`).

- Written **once**; not reconciled like extracted facts.
- May **age out** faster than derived beliefs while still leaving supersession history intact.
- Browsed with session tooling – see [Sessions and turns](/docs/spectron/mental-model/sessions-and-turns).

## 2. Identity – who the principal is

Stable biographical and organisational facts: name, role, employer, durable preferences. **Long retention**, low decay, high retrieval boost toward prompt assembly.

## 3. Knowledge – what the principal knows

Project facts, observations, and references the user has **shared or learned**, distinct from curated documents. **Medium retention** – decays without reinforcement unless consolidated.

## 4. Context – what is happening now

Active topics, intents, and the **working set** for the current conversation. **Short retention**, replaced rapidly, anchors the next turn.

## 5. Instructions – how to behave

Behavioural directives (“British English”, “never use my first name”). Stored in a dedicated **`instruction`** path because they govern **behaviour**, not factual retrieval weights – applied when prompts are assembled.

## 6. Uncertainty – explicit gaps

Rows emitted when the reconciler is not confident enough to supersede, when provenances contradict, or when extraction cannot ground a reference. **Uncertainty** is surfaced to agents and users so gaps are visible instead of hallucinated over.

> Naming note: older docs called this category **Unknowns**. **`uncertainty`** rows are the concrete substrate representation today.

## At a glance

| Category | Holds | Typical lifetime |
| --- | --- | --- |
| Episodic | Raw turns / transcripts | Short-to-medium; operational TTLs |
| Identity | Stable facts about the principal | Long |
| Knowledge | Learned / shared factual context | Medium; reinforce or decay |
| Context | Current working state | Short |
| Instructions | Behaviour preferences | Until revoked |
| Uncertainty | Deliberate “we do not know” | Until resolved or superseded |

## Implementation note

Exact table names, decay coefficients, and retrieval boosts may evolve between releases. The **invariants** documented here match the Spectron architecture narrative (`doc/spectron.md`): six experiential categories plus episodic raw material, distinct from trace memory and from authoritative document streams.
