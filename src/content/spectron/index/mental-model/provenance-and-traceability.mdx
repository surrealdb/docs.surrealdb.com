# Provenance and traceability

Spectron‚Äôs accuracy story depends on one invariant: **no fact-bearing row is anonymous**. Provenance is a **structured field**, not an afterthought in application logs.

## The `source` object (conceptual)

Every fact-bearing row (`entity`, `attribute`, `relation`, `instruction`, `uncertainty`, corrections, memory chunks, ‚Ä¶) carries a **`source`** object. Exact JSON shape evolves with releases; the architecture contract includes:

| Field | Role |
| --- | --- |
| `source.kind` | `turn`, `document`, `upsert`, `reflect`, `elaboration`, `consolidation`, ‚Ä¶ |
| `source.ref` | Originating turn, document, or trace identifier |
| `source.session_id`, `source.turn_at` | When the row came from conversation |
| `source.valid_from`, `source.span: { start, end }` | Valid-time anchor and **lexical attribution** into the originating bytes |
| `source.location` | Optional geometry for where the fact was **captured** (distinct from where it applies) |
| `source.trust` | Source prior ‚Äì admin documents rank above casual turns; reflections and elaborations rank lower by default |
| `source.derived_from` | Lineage vector for reflections, elaborations, consolidations, collective promotion |

Older examples in this documentation may still show a flat `source_turn` field in JSON snippets ‚Äì think of that as a **projection** of `source.ref` for conversational rows. New material should prefer the `source` object vocabulary from `doc/spectron.md`.

## Why spans matter

`source.span` stores **byte offsets** into the originating turn or document passage. Citations are therefore **stored data**, not best-effort model prose. That powers lexical coherence, audit, and UI ‚Äújump to quote‚Äù experiences ‚Äì see [Coherence, retrieval, and cost tiers](/docs/spectron/architecture/coherence-retrieval-and-tiers).

## Traces complete the story

Extraction and reconciliation emit **`decision_trace`** nodes with edges to candidates considered, rows created or superseded, and parent traces. Ranked reads emit **`retrieval_trace`**; `/chat` and `/reflect` emit **`response_trace`**. Together they let you answer ‚Äú**what bytes produced this belief?**‚Äù and ‚Äú**which retrieval path led to this answer?**‚Äù without a separate observability SKU ‚Äì see [Traces and memory evolution](/docs/spectron/architecture/traces-and-evolution).
