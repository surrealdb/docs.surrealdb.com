# Spectron

Spectron is an accurate, observable, conversational knowledge system for AI agents. It gives agents a persistent memory layer — backed by SurrealDB — where facts are structured, corrections are tracked, and every piece of knowledge traces back to the conversation that produced it.

The core design question Spectron answers is: **can someone watch the memory state in real-time and verify that the system is correct?** That means no black-box embeddings, no silently stale facts, no untraceable hallucinations creeping into context.

## Two layers of knowledge

Spectron organises memory into two distinct layers:

| Layer | Name | Contents |
|-------|------|----------|
| Layer 0 | Knowledge | Authoritative organisational data — documents, policies, product information, FAQs. Ingested via file upload or connectors. |
| Layer 1 | Memory | Conversational memory — entities, attributes, relations, and instructions learned from agent sessions and user interactions. |

Layer 0 is the source of truth. When Layer 1 contradicts Layer 0, Layer 0 wins. This authority hierarchy prevents conversational drift from corrupting authoritative data.

## How it fits together

Every conversation is a **session**, divided into **turns**. Each turn runs a synchronous pipeline: classify the input, extract structured facts, reconcile against existing memory, persist to SurrealDB. Retrieval uses tiered query resolution — from direct graph lookup through to full LLM synthesis — so the simplest queries never touch a language model.

Memory is isolated into **Contexts**. Each Context is an independent `(namespace, database)` pair in SurrealDB, suitable for separate products, tenants, or teams. Every piece of data carries dimensional scope attributes — organisation, user, project — enabling per-user isolation and collaborative memory within a single Context.
