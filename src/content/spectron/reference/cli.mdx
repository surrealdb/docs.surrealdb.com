# CLI reference

Spectron ships two command-line tools:

- **`spectron`** ‚Äì server lifecycle commands for starting and bootstrapping a Spectron instance
- **`spectronctl`** ‚Äì operator interface for managing contexts, keys, and knowledge

Both tools are included in the Spectron server distribution and the Python package.

```bash
pip install spectron
```

---

## Server management

### spectron init

Bootstraps a fresh Spectron instance by creating the first management API key. Run this once after installing Spectron for the first time.

```
spectron init --admin-key <key>
```

**Flags**

| Flag | Description |
|-------------|--------------------------------------|
| `--admin-key` | The management key to register. Must be at least 32 characters. |
| `--db-url` | SurrealDB connection URL (default: `ws://localhost:8000`) |
| `--db-ns` | SurrealDB namespace (default: `spectron`) |
| `--db-db` | SurrealDB database name (default: `spectron`) |

**Example**

```bash
spectron init --admin-key "mgmt-$(openssl rand -hex 32)"
```

---

### spectron start

Starts the Spectron server process.

```
spectron start [flags]
```

**Flags**

| Flag | Default | Description |
|-------------|---------|--------------------------------------|
| `--port` | `4000` | HTTP port to listen on |
| `--host` | `0.0.0.0` | Host to bind to |
| `--db-url` | `ws://localhost:8000` | SurrealDB connection URL |
| `--db-ns` | `spectron` | SurrealDB namespace |
| `--db-db` | `spectron` | SurrealDB database name |
| `--log-level` | `info` | Log verbosity: `debug`, `info`, `warn`, `error` |
| `--config` | ‚Äì | Path to a YAML config file |

**Example**

```bash
spectron start --port 4000 --db-url ws://surreal:8000
```

Configuration can also be provided via environment variables prefixed with `SPECTRON_`:

```bash
SPECTRON_PORT=4000 SPECTRON_DB_URL=ws://surreal:8000 spectron start
```

## Context management

All `spectronctl contexts` commands require an admin or management API key, provided via `--api-key` or the `SPECTRON_API_KEY` environment variable.

### spectronctl contexts list

Lists all contexts registered in the instance.

```
spectronctl contexts list [flags]
```

**Flags**

| Flag | Description |
|-----------|-------------------------------|
| `--api-key` | Management API key |
| `--server` | Spectron server URL (default: `http://localhost:4000`) |
| `--json` | Output as JSON |

**Example**

```bash
spectronctl contexts list --api-key mgmt-‚Ä¶
```

```
ID          NAME        CREATED
ctx_acme    acme        2026-01-15T09:00:00Z
ctx_demo    demo        2026-02-10T14:30:00Z
```

---

### spectronctl contexts create

Creates a new context.

```
spectronctl contexts create <name> [flags]
```

**Arguments**

| Argument | Description |
|----------|---------------------------|
| `name` | Unique name for the context |

**Flags**

| Flag | Description |
|-------------|-----------------------------------|
| `--api-key` | Management API key |
| `--description` | Human-readable description |
| `--json` | Output created context as JSON |

**Example**

```bash
spectronctl contexts create acme --description "Acme Corp production context"
```

---

### spectronctl contexts delete

Deletes a context and all associated memory, sessions, and keys. This operation is irreversible.

```
spectronctl contexts delete <context-id> [flags]
```

**Flags**

| Flag | Description |
|-------------|---------------------------|
| `--api-key` | Management API key |
| `--confirm` | Skip the confirmation prompt |

**Example**

```bash
spectronctl contexts delete ctx_demo --confirm
```

## Key management

### spectronctl contexts keys list

Lists all API keys for a context.

```
spectronctl contexts keys list <context-id> [flags]
```

**Example**

```bash
spectronctl contexts keys list ctx_acme --api-key mgmt-‚Ä¶
```

```
ID          NAME          PRINCIPAL   SCOPE FLOOR               EXPIRES
key_01hw‚Ä¶   planner       agent       org=acme, agent=planner   never
key_02ab‚Ä¶   tool-search   agent       org=acme, agent=planner,  2026-06-01
                                      tool=search
```

---

### spectronctl contexts keys create

Creates a new API key for a context.

```
spectronctl contexts keys create <context-id> <name> [flags]
```

**Flags**

| Flag | Description |
|-----------------|---------------------------------------------|
| `--api-key` | Management API key |
| `--principal` | Key principal type: `agent`, `supervisor`, `management` |
| `--scope-floor` | JSON-encoded scope floor, e.g. `'{"org":"acme","agent":"planner"}'` |
| `--expires-at` | Expiry timestamp in RFC 3339 format |
| `--json` | Output created key as JSON (includes plaintext key ‚Äì shown once only) |

**Example**

```bash
spectronctl contexts keys create ctx_acme planner \
    --principal agent \
    --scope-floor '{"org":"acme","agent":"planner"}' \
    --api-key mgmt-‚Ä¶ \
    --json
```

> The plaintext key is returned only at creation time. Store it immediately.

---

### spectronctl contexts keys delete

Deletes (revokes) an API key. Active requests using the key will fail immediately.

```
spectronctl contexts keys delete <context-id> <key-id> [flags]
```

**Example**

```bash
spectronctl contexts keys delete ctx_acme key_02ab‚Ä¶ --api-key mgmt-‚Ä¶
```

## Knowledge operations

### spectronctl contexts ingest

Bulk ingests a file into a context's authoritative knowledge base.

```
spectronctl contexts ingest <context-id> <file> [flags]
```

**Arguments**

| Argument | Description |
|--------------|-------------------------------------------|
| `context-id` | Target context |
| `file` | Path to a local file (PDF, DOCX, TXT, MD) |

**Flags**

| Flag | Description |
|--------------|----------------------------------------------|
| `--api-key` | Management API key |
| `--label` | Human-readable label for the ingested document |
| `--tags` | Comma-separated tags |
| `--watch` | Stream ingestion progress to stdout |

**Example**

```bash
spectronctl contexts ingest ctx_acme ./product-catalogue.pdf \
    --label "Product Catalogue Q1 2026" \
    --tags "products,catalogue" \
    --watch
```

For bulk ingestion of directories, use a shell glob:

```bash
for f in ./docs/*.pdf; do
    spectronctl contexts ingest ctx_acme "$f" --api-key mgmt-‚Ä¶
done
```

---

### spectronctl contexts knowledge query

Runs a knowledge query against a context's authoritative knowledge base from the command line. Useful for quickly verifying ingestion results or debugging retrieval.

```
spectronctl contexts knowledge query <context-id> <query> [flags]
```

**Arguments**

| Argument | Description |
|--------------|---------------------------|
| `context-id` | Context to query |
| `query` | Natural language query string |

**Flags**

| Flag | Default | Description |
|---------|---------|-------------------------------|
| `--api-key` | ‚Äì | Management API key |
| `--top-k` | `5` | Number of results to return |
| `--json` | ‚Äì | Output results as JSON |

**Example**

```bash
spectronctl contexts knowledge query ctx_acme \
    "What is the return policy for AirPods Pro?" \
    --top-k 3 \
    --api-key mgmt-‚Ä¶
```

```
[1] product-catalogue.pdf (score: 0.94)
    AirPods Pro carry a 30-day no-fault return policy and a 12-month
    limited warranty covering manufacturing defects.

[2] support-faq.pdf (score: 0.81)
    To initiate a return, visit support.example.com/returns ‚Ä¶
```

## Global flags

All `spectronctl` commands accept these global flags:

| Flag | Environment variable | Description |
|-------------|----------------------|-------------------------------|
| `--api-key` | `SPECTRON_API_KEY` | API key for authentication |
| `--server` | `SPECTRON_SERVER` | Spectron server URL |
| `--json` | ‚Äì | Force JSON output |
| `--quiet` | ‚Äì | Suppress informational output |
| `--help` | ‚Äì | Show help for any command |

## Shell completion

Enable tab completion for your shell:

```bash
# bash
spectronctl completion bash >> ~/.bashrc

# zsh
spectronctl completion zsh >> ~/.zshrc

# fish
spectronctl completion fish > ~/.config/fish/completions/spectronctl.fish
```
