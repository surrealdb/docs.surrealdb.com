# What is SurrealDB Cloud?


SurrealDB Cloud transforms the database experience, providing the power and versatility of SurrealDB without the complexity of managing infrastructure. Whether you’re building for passion projects or enterprise-scale applications, SurrealDB Cloud offers the scalability, resilience, and innovation you need to stay ahead.

Enjoy a fully managed solution that takes the hassle out of infrastructure operations. With SurrealDB’s ACID compliance, you can confidently run transactions at scale across multiple environments, ensuring data integrity and reliability.

Designed to support your growth, SurrealDB Cloud scales effortlessly [from prototypes to large-scale deployments](/docs/build/deployment/surrealdb-cloud/getting-started/create-an-instance). By leveraging centralised data, it enables departments or teams to work efficiently while maintaining consistent data integrity and reducing redundancy.

Our transparent and [predictable pricing](/pricing) ensures you only pay for what you need.

## Cloud plans

SurrealDB Cloud spans single-node and multi-node managed deployments:

- **Start** — one compute node with dedicated storage; scale vertically by resizing the instance. Includes a free tier for getting started. Suited to development, staging, and production where a single server is acceptable.
- **Scale** — multi-node clusters on [SurrealDS](https://surrealdb.com/platform/surrealds), starting at three compute units, for production workloads that must survive node failure and grow query capacity horizontally. SurrealDB Cloud operates the cluster — provisioning, HA, patching, backups, and upgrades, allowing you to focus on applications instead of platform engineering.

See [Cloud architecture](/docs/manage/cloud/architecture) for diagrams and [Deployment](/docs/build/deployment) for how Cloud fits alongside self-hosted and embedded options.

## What can you do?

When using SurrealDB Cloud, you can connect to your SurrealDB instance with a few clicks. You can connect to any SurrealDB instance via Surrealist, the command-line interface, and any of the supported SDKs. 


<Tabs synckey="What can you do">
<TabItem label="Create an instance">
	SurrealDB Cloud allows you to connect to your SurrealDB instance with just a few clicks. You can create an instance, connect to it, and start building your application.

	**1. Create Instance.**

	When creating an instance, you will enter the instance name and organisation to ensure that you can easily identify your instance. Finally, you can select the version of SurrealDB to use and the region to deploy your instance.

	**2. Select a region.**

	Once you have created your instance, you can select the region to deploy your instance. This will allow you to connect to your instance from anywhere in the world.

	**3. Select an Instance preset.**

	Once you have selected a region, you can select an instance preset. This will allow you to select the instance size and type to best suit your needs.

	**4. Finalise and connect to your Instance.**

	Once you have created your instance, you can connect to it from anywhere.
</TabItem>
<TabItem label="Connect to your instance">
	You can connect to any SurrealDB Cloud Instance via [Surrealist](/docs/build/deployment/surrealdb-cloud/connecting/via-surrealist), the [command-line interface (CLI)](/docs/build/deployment/surrealdb-cloud/connecting/via-cli), and [any of the supported SDKs](/docs/build/deployment/surrealdb-cloud/connecting/via-sdk).

	[Connect via Surrealist](/docs/build/deployment/surrealdb-cloud/connecting/via-surrealist)

	Since SurrealDB Cloud is closely integrated with Surrealist, you can execute queries directly from the interface. This allows you to interact with your data in real time, without having to switch between different tools.

	[Connect via CLI](/docs/build/deployment/surrealdb-cloud/connecting/via-cli)

	If you prefer to use a command-line interface, you can use the SurrealDB CLI to connect to your instance. This connection includes a token that is used to authenticate your connection.

	To do so, select the connect with the CLI option and copy the command.

	[Connect via SDK](/docs/build/deployment/surrealdb-cloud/connecting/via-sdk)

	You can also connect to your instance using any of the supported SDKs. In your preferred SDK, use the connect method and pass in your instance endpoint.
</TabItem>
</Tabs>
