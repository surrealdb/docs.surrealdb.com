# Data manipulation

The Go SDK provides generic top-level functions for common CRUD operations on records and tables. These functions work with [`*DB`](/docs/sdk/golang/api/core/db), [`*Session`](/docs/sdk/golang/api/core/session), and [`*Transaction`](/docs/sdk/golang/api/core/transaction) through the [`sendable`](/docs/sdk/golang/api/types#sendable) constraint, and return typed results through Go generics.

This page covers how to target tables and records, and how to select, create, insert, update, merge, patch, and delete data.

## Targeting tables and records

Most data manipulation functions accept a `what` parameter that determines the scope of the operation. You can pass a [`Table`](/docs/sdk/golang/api/values/table) to target all records in a table, or a [`RecordID`](/docs/sdk/golang/api/values/record-id) to target a specific record.

```go
import "github.com/surrealdb/surrealdb.go/pkg/models"

all, err := surrealdb.Select[[]Person](ctx, db, models.Table("persons"))

one, err := surrealdb.Select[Person](ctx, db, models.NewRecordID("persons", "tobie"))
```

When a [`Table`](/docs/sdk/golang/api/values/table) is passed, operations that return data return a slice. When a [`RecordID`](/docs/sdk/golang/api/values/record-id) is passed, they return a single value. Use the appropriate type parameter to match.

## Selecting records

[`Select`](/docs/sdk/golang/api/core/db#select) retrieves records from the database. Pass a [`Table`](/docs/sdk/golang/api/values/table) to get all records, or a [`RecordID`](/docs/sdk/golang/api/values/record-id) to get a single record.

```go
persons, err := surrealdb.Select[[]Person](ctx, db, models.Table("persons"))
if err != nil {
	log.Fatal(err)
}

tobie, err := surrealdb.Select[Person](ctx, db, models.NewRecordID("persons", "tobie"))
if err != nil {
	log.Fatal(err)
}
```

## Creating records

[`Create`](/docs/sdk/golang/api/core/db#create) creates a new record. Pass a [`Table`](/docs/sdk/golang/api/values/table) to generate a random ID, or a [`RecordID`](/docs/sdk/golang/api/values/record-id) to specify the ID explicitly. The data can be a struct or a map.

```go
person, err := surrealdb.Create[Person](ctx, db, models.Table("persons"), Person{
	Name:    "Tobie",
	Surname: "Morgan Hitchcock",
})

specific, err := surrealdb.Create[Person](ctx, db, models.NewRecordID("persons", "tobie"), map[string]any{
	"name":    "Tobie",
	"surname": "Morgan Hitchcock",
})
```

## Inserting records

[`Insert`](/docs/sdk/golang/api/core/db#insert) inserts one or more records into a table. This is useful for bulk operations.

```go
persons, err := surrealdb.Insert[Person](ctx, db, models.Table("persons"), []map[string]any{
	{"name": "Alice", "age": 30},
	{"name": "Bob", "age": 25},
})
```

## Creating relations

The SDK provides two ways to create graph edges between records.

[`Relate`](/docs/sdk/golang/api/core/db#relate) creates a relation with an auto-generated ID. The [`Relationship.ID`](/docs/sdk/golang/api/types#relationship) field is ignored.

```go
rel, err := surrealdb.Relate[map[string]any](ctx, db, &surrealdb.Relationship{
	In:       models.NewRecordID("persons", "tobie"),
	Out:      models.NewRecordID("posts", "first"),
	Relation: models.Table("wrote"),
	Data:     map[string]any{"created_at": "2026-01-01T00:00:00Z"},
})
```

[`InsertRelation`](/docs/sdk/golang/api/core/db#insertrelation) works like [`Insert`](/docs/sdk/golang/api/core/db#insert) but for relation tables. It allows you to specify the ID explicitly via the [`Relationship.ID`](/docs/sdk/golang/api/types#relationship) field.

```go
rel, err := surrealdb.InsertRelation[map[string]any](ctx, db, &surrealdb.Relationship{
	In:       models.NewRecordID("persons", "tobie"),
	Out:      models.NewRecordID("posts", "first"),
	Relation: models.Table("wrote"),
})
```

## Replacing records

[`Update`](/docs/sdk/golang/api/core/db#update) replaces the entire content of a record or all records in a table. Fields not included in the new data are removed.

```go
updated, err := surrealdb.Update[Person](ctx, db, models.NewRecordID("persons", "tobie"), Person{
	Name:    "Tobie",
	Surname: "Morgan Hitchcock",
})
```

> [!NOTE]
> Because [`Update`](/docs/sdk/golang/api/core/db#update) performs a full replacement, omitted fields are deleted from the record. Use [`Merge`](/docs/sdk/golang/api/core/db#merge) if you want to preserve existing fields.

## Upserting records

[`Upsert`](/docs/sdk/golang/api/core/db#upsert) creates a record if it does not already exist, or replaces it entirely if it does.

```go
person, err := surrealdb.Upsert[Person](ctx, db, models.NewRecordID("persons", "tobie"), Person{
	Name:    "Tobie",
	Surname: "Morgan Hitchcock",
})
```

## Merging data

[`Merge`](/docs/sdk/golang/api/core/db#merge) deep-merges the provided data into the existing record, preserving fields not mentioned in the merge payload.

```go
merged, err := surrealdb.Merge[Person](ctx, db, models.NewRecordID("persons", "tobie"), map[string]any{
	"age": 35,
})
```

## Applying patches

[`Patch`](/docs/sdk/golang/api/core/db#patch) applies [JSON Patch (RFC 6902)](https://jsonpatch.com/) operations to a record or all records in a table. Each operation is a [`PatchData`](/docs/sdk/golang/api/types#patchdata) with `Op`, `Path`, and `Value` fields.

```go
patches, err := surrealdb.Patch(ctx, db, models.NewRecordID("persons", "tobie"), []surrealdb.PatchData{
	{Op: "replace", Path: "/surname", Value: "Hitchcock"},
	{Op: "add", Path: "/verified", Value: true},
})
```

Supported operations include `add`, `remove`, `replace`, `move`, `copy`, and `test`.

## Deleting records

[`Delete`](/docs/sdk/golang/api/core/db#delete) removes a specific record or all records from a table. The function returns the deleted record(s).

```go
deleted, err := surrealdb.Delete[Person](ctx, db, models.NewRecordID("persons", "tobie"))

allDeleted, err := surrealdb.Delete[[]Person](ctx, db, models.Table("persons"))
```
